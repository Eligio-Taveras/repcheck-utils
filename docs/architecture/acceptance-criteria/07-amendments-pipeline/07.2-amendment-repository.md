# 7.2 Amendment Repository

> Part of [Component 7 — Amendments Pipeline](../07-AMENDMENTS-PIPELINE.md)

## Overview

Doobie repository for the `amendments` table. Simple upsert on natural key — no history archival, no child tables. Uses `UpsertHelper` from `ingestion-common` (Component 3 §3.5) for SQL generation.

All table and column names come from `Tables.*` constants in `pipeline-models` (Component 2). `AmendmentDO` comes from `shared-models` (Component 1 §1.2).

## Classes

### `AmendmentRepository[F[_]]` — Trait + Impl

```
trait AmendmentRepository[F[_]] extends EntityRepository[F, AmendmentDO] {
  def upsert(amendment: AmendmentDO): F[Unit]
  def findByAmendmentId(amendmentId: String): F[Option[AmendmentDO]]
  def findByBillId(billId: String): F[List[AmendmentDO]]
  def findByCongress(congress: Int): F[List[AmendmentDO]]
  def insertIfNotExists(entity: AmendmentDO): F[Unit]  // from EntityRepository
}
```

> **Extends `EntityRepository[F, AmendmentDO]`** — this enables `PlaceholderCreator` (Component 3 §3.6) to create placeholder amendments if a future pipeline references an amendment not yet ingested. Currently no pipeline creates amendment placeholders, but the interface is pre-wired for consistency with `MemberDO` and `BillDO`.

**`upsert`:**
- Uses `UpsertHelper.upsertFragment` with `Tables.Amendments`
- Conflict column: `amendment_id` (natural key: `{congress}-{amendmentType}-{number}`)
- Updates all non-key, non-audit columns on conflict
- Sets `updated_at = NOW()` on conflict

**`findByAmendmentId`:**
- Simple SELECT by `amendment_id`
- Returns `Option[AmendmentDO]` — `None` if not found
- Used by `AmendmentProcessor` to fetch stored amendment for `ChangeDetector` comparison

**`findByBillId`:**
- SELECT by `bill_id`
- Returns all amendments associated with a bill
- Used by downstream consumers (e.g., bill re-analysis pipeline) to fetch amendments for a bill

**`findByCongress`:**
- SELECT by `congress`
- Returns all amendments for a given congress
- Ordered by `amendment_id`

**`insertIfNotExists`:**
- `INSERT ... ON CONFLICT DO NOTHING` with the provided `AmendmentDO`
- Inherited from `EntityRepository` — used by `PlaceholderCreator` pattern
- No-op if amendment already exists

### `AmendmentDO` Field Mapping

Per Component 1 §1.2, `AmendmentDO` maps to the `amendments` table:

| DO Field | Column | Type | Notes |
|----------|--------|------|-------|
| `amendmentId` | `amendment_id` | String (PK) | Natural key: `{congress}-{amendmentType}-{number}` |
| `congress` | `congress` | Int | |
| `amendmentType` | `amendment_type` | String | `"HAMDT"`, `"SAMDT"`, or `"SUAMDT"` |
| `number` | `number` | String | Amendment number within congress |
| `billId` | `bill_id` | Option[String] | FK → `bills.bill_id`. Some amendments are standalone |
| `chamber` | `chamber` | String | `"House"` or `"Senate"` |
| `description` | `description` | Option[String] | |
| `purpose` | `purpose` | Option[String] | |
| `sponsorBioguideId` | `sponsor_bioguide_id` | Option[String] | FK → `members.member_id`. Some amendments have no identified sponsor |
| `submittedDate` | `submitted_date` | Option[Instant] | |
| `latestActionDate` | `latest_action_date` | Option[Instant] | |
| `latestActionText` | `latest_action_text` | Option[String] | |

## Acceptance Criteria

| Criterion | Test |
|-----------|------|
| `upsert` inserts new amendment | Insert → `findByAmendmentId` → returns amendment with all fields |
| `upsert` updates existing amendment on conflict | Insert → upsert with changed description → verify description updated |
| `upsert` does not create duplicate rows | Upsert same `amendmentId` twice → exactly 1 row |
| `upsert` sets `updated_at` on conflict | Upsert existing → `updated_at` changed |
| `findByAmendmentId` returns `None` for missing amendment | Unknown ID → `None` |
| `findByAmendmentId` returns stored amendment with all fields | Insert → find → all fields match |
| `findByBillId` returns amendments for a bill | Insert 3 amendments for bill X → `findByBillId(X)` → returns 3 |
| `findByBillId` returns empty list when no amendments | Unknown bill → empty list |
| `findByCongress` returns amendments for a congress | Insert amendments for congress 117 and 118 → `findByCongress(117)` → returns only 117 |
| `insertIfNotExists` inserts when missing | New amendment → inserted |
| `insertIfNotExists` is no-op when exists | Existing amendment → row unchanged |
| `insertIfNotExists` does not overwrite richer data | Full amendment exists → `insertIfNotExists` with placeholder → full data preserved |
| FK constraint on `bill_id` requires bill to exist | Amendment with unknown `bill_id` → FK violation |
| FK constraint on `sponsor_bioguide_id` requires member to exist | Amendment with unknown sponsor → FK violation |
| `bill_id = None` is valid (standalone amendment) | Insert amendment without bill → succeeds |
| `sponsor_bioguide_id = None` is valid | Insert amendment without sponsor → succeeds |
| All SQL uses `Tables.*` constants | No hardcoded table/column name strings |
