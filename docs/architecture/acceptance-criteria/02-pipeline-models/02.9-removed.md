# ~~2.9 Deferred Entity Resolution~~ — REMOVED

> **Superseded by**: Placeholder Entity Pattern (Component 3 §3.6).
>
> The `unresolved_references` table and resolution flow are no longer needed. Instead, when a pipeline encounters a cross-entity reference to a not-yet-ingested entity, it creates a **placeholder row** in the target entity's table with only the natural key populated. The owning pipeline fills in full data on its next run via normal upsert + `ChangeDetector` diff. No separate tracking table required.
