# 2.7 Launcher Execution Model

> **Design rationale:** The Launcher is a Cloud Run Job triggered on a frequent schedule (e.g., every 5 minutes). It pulls Pub/Sub messages, resolves workflows, checks dependencies, resolves macros, and launches target applications via the Cloud Run Jobs API. The Launcher never writes to `workflow_run_steps` — applications update their own state.

### Launcher Traits

| Trait | Methods | Notes |
|-------|---------|-------|
| `MessagePuller[F[_]]` | `pull(subscriptionId: String, maxMessages: Int): F[List[PipelineEvent[String]]]` | Pulls raw JSON messages from Pub/Sub. Returns deserialized envelopes with raw payload strings. |
| `WorkflowResolver[F[_]]` | `resolve(gcsPath: String): F[WorkflowDefinition]` | Reads and parses a workflow definition from GCS. |
| `DependencyChecker[F[_]]` | `allDependenciesMet(workflowRunId: UUID, dependencies: List[String]): F[Boolean]` | Reads `workflow_run_steps` (read-only) to check if all dependency `stepName` values are in `Completed` status. `dependencies` is a list of `stepName` strings (matching `workflow_run_steps.step_name`). |
| `MacroResolver` | `resolve(template: String, context: MacroContext): String` | Replaces macro tokens (`{{run_id}}`, `{{message.billId}}`, etc.) with runtime values. |
| `JobLauncher[F[_]]` | `launch(config: ResolvedStepConfig): F[LaunchResult]` | Calls the Cloud Run Jobs API with the fully resolved container configuration. |

### Supporting Types

| Class | Fields | Notes |
|-------|--------|-------|
| `MacroContext` | runId: UUID, timestamp: Instant, date: String, messagePayload: Map[String, String] | Values available for macro resolution. `messagePayload` is flattened from the triggering Pub/Sub message. |
| `ResolvedStepConfig` | stepName: String, container: ContainerConfig (with macros resolved), resources: ResourceConfig, execution: ExecutionConfig, networking: NetworkConfig, volumes: VolumeConfig, identity: IdentityConfig, healthChecks: HealthCheckConfig, metadata: MetadataConfig | Fully resolved step configuration ready for Cloud Run Jobs API. |
| `LaunchResult` | success: Boolean, jobName: Option[String], errorMessage: Option[String] | Result of a Cloud Run Job launch attempt. |

### Launcher Flow (pseudocode)

```
for each message in pull(subscription):
  event = decode(message)
  workflow = resolve(event → workflow GCS path)
  step = workflow.steps.find(_.triggeredBy contains event.eventType)
  if step is None:
    log warning, ack message, continue
  if not allDependenciesMet(runId, step.dependencies):
    nack message (will be redelivered on next tick)
    continue
  context = MacroContext(runId, now, today, event.payload fields)
  resolvedConfig = resolve macros in step.container using context
  result = launch(resolvedConfig)
  if result.success:
    ack message
  else:
    nack message (will be redelivered)
```

### Acceptance Criteria — 2.7 Launcher Execution Model

**Functional:**
1. `MessagePuller` pulls messages from Pub/Sub and deserializes them as `PipelineEvent[String]` (raw JSON payload).
2. `WorkflowResolver` reads a workflow definition from GCS and returns a parsed `WorkflowDefinition`.
3. `DependencyChecker` reads `workflow_run_steps` and returns `true` only if ALL dependency steps have status `Completed`.
4. `MacroResolver` replaces all `{{...}}` tokens in a string with values from `MacroContext`.
5. `MacroResolver` supports nested message payload access: `{{message.billId}}` extracts `billId` from the triggering message.
6. `MacroResolver` returns the original token unchanged (not empty string) if a macro has no matching value — this makes unresolved macros visible for debugging.
7. `JobLauncher` calls the Cloud Run Jobs API with the full resolved configuration including container, resources, networking, volumes, identity, and health checks.
8. `JobLauncher` passes `secretMounts` as Cloud Run secret references — never reads secret values.
9. The Launcher acks a Pub/Sub message only after successful launch. Failed launches result in nack (redelivery).
10. The Launcher nacks messages whose dependencies are not yet met (redelivered on next tick).
11. The Launcher logs each launch attempt with: step name, workflow name, run ID, event type, and result.

**Error Handling:**
12. If `WorkflowResolver` fails to read or parse the GCS file, the message is nacked and an error is logged with the GCS path.
13. If `JobLauncher` fails (Cloud Run API error), the message is nacked and the error is logged with the step name and API response.
14. If no step matches the event type in the workflow, the message is acked (not an error — the event may not trigger any step in this workflow) and a warning is logged.

**Test Requirements:**
15. Test `DependencyChecker` with all dependencies met — verify returns `true`.
16. Test `DependencyChecker` with one dependency not completed — verify returns `false`.
17. Test `DependencyChecker` with no dependencies (root step) — verify returns `true`.
18. Test `MacroResolver` with `{{run_id}}` — verify replaced with UUID string.
19. Test `MacroResolver` with `{{message.billId}}` — verify extracted from message payload.
20. Test `MacroResolver` with unknown macro `{{unknown}}` — verify token left unchanged.
21. Test full Launcher flow with mock traits: message → resolve → check deps → resolve macros → launch → ack.
22. Test Launcher flow with unmet dependencies — verify nack, no launch.
23. Test Launcher flow with launch failure — verify nack, error logged.
