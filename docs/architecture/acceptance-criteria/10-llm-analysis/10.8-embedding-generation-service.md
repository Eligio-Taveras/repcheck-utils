# Area 10.8 — Embedding Generation Service

> Project: Shared (both pipelines) | Package: `repcheck.analysis.embedding`

## Purpose

Local DJL/ONNX utility service for generating search embeddings. Called at multiple pipeline stages — not a post-processing step. All embeddings are generated locally with zero API cost.

---

## Classes

### `SemanticEmbeddingService[F[_]]` — Local Embedding Generation (10.8)
**Package:** `repcheck.analysis.embedding`

Generates search-quality embeddings using a local ONNX model via DJL. Used by both the decomposition pipeline (section and concept group embeddings) and the analysis pipeline (finding embeddings).

```scala
class SemanticEmbeddingService[F[_]: Async](
  predictor: Predictor[String[], float[][]],
  config: EmbeddingConfig
)
```

**Factory:**
```scala
object SemanticEmbeddingService {
  def resource[F[_]: Async](config: EmbeddingConfig): Resource[F, SemanticEmbeddingService[F]]
}
```

The `resource` factory loads the ONNX model at startup and releases it on shutdown.

#### `embedBatch(texts: List[String]): F[List[Array[Float]]]`

1. Validate `texts` is non-empty — return empty list if empty
2. Chunk `texts` into batches of `config.batchSize`
3. For each batch:
   a. Convert to `String[]` array
   b. Execute `Async[F].blocking(predictor.predict(textArray))`
   c. Convert `float[][]` result to `List[Array[Float]]`
4. Concatenate all batch results
5. Validate output dimensionality: each vector must be `config.expectedDimension` elements
6. On DJL failure → raise `EmbeddingGenerationFailed(error.getMessage, texts.size)`

#### `embedSingle(text: String): F[Array[Float]]`

1. Delegate to `embedBatch(List(text))`
2. Return single result

#### `embedFindings(passOutputs: List[ConceptGroupPassOutput]): F[List[(UUID, Array[Float])]]`

1. Flatten all findings from `passOutputs`: for each `ConceptGroupPassOutput`, extract findings from `stance`, `pork`, `impact` output schemas
2. Extract finding text (the `summary` field from each finding) and finding IDs
3. Call `embedBatch(findingTexts)`
4. Zip finding IDs with embeddings
5. Return `List[(UUID, Array[Float])]` ready for persistence via `persister.persistFindingEmbeddings`

> **`ConceptGroupPassOutput` dependency:** This method takes the type defined in 10.6. The `embedBatch` method is type-agnostic (takes `List[String]`), but `embedFindings` is a convenience method that understands the analysis output structure.

**Callers and their inputs:**
- **Decomposition pipeline (10.4):**
  - `embedBatch(sections.map(_.content))` → `bill_text_sections.embedding`
  - `embedBatch(simplifiedTexts)` → `bill_concept_groups.embedding`
- **Analysis pipeline (10.6/10.7):**
  - `embedFindings(passOutputs)` → `bill_findings.embedding`

### `EmbeddingConfig` — Service Config (10.8)
**Package:** `repcheck.analysis.embedding`

```scala
case class EmbeddingConfig(
  modelName: String,
  batchSize: Int,
  expectedDimension: Int,
  modelPath: Option[String] = None
)
```

- `modelName` — ONNX model identifier (e.g., a model producing 1536-dim or 768-dim vectors)
- `batchSize` — max texts per DJL batch call (balances memory vs. throughput)
- `expectedDimension` — expected vector dimension. Must match the `vector(N)` column dimension in AlloyDB schema. Validated at runtime.
- `modelPath` — optional local path override for testing

> **Dimension alignment:** The `expectedDimension` config value must match both the ONNX model's output and the `vector(N)` column size in AlloyDB. If the model produces 768-dim vectors, set `expectedDimension = 768` and ensure the schema migration uses `vector(768)`. A mismatch will be caught at runtime during the validation step in `embedBatch`.

### Error Types (10.8)
**Package:** `repcheck.analysis.embedding`

```scala
case class EmbeddingGenerationFailed(
  reason: String,
  batchSize: Int
) extends Exception(s"Embedding generation failed for batch of $batchSize: $reason")
```

> **Shared error type.** Both `SectionEmbedder` (10.5) and `SemanticEmbeddingService` (10.8) raise `EmbeddingGenerationFailed`. The error lives in the shared `repcheck.analysis.embedding` package, which both pipelines depend on. This avoids the decomposition pipeline depending on `repcheck.decomposition.pipeline.errors` for a shared type.

## Key Design Decisions

1. **Single service, multiple callers.** Both pipelines use the same `SemanticEmbeddingService`. In the SBT build, this is either a shared internal project or duplicated in each pipeline's classpath.
2. **Local only, zero API cost.** All embeddings are generated in-process via DJL/ONNX. No Claude, OpenAI, or other embedding API calls. This is critical for cost control — embedding generation scales with bill count.
3. **Separate from clustering embeddings.** The `SectionEmbedder` (10.5) generates 384-dim ephemeral vectors for clustering. `SemanticEmbeddingService` generates search-quality vectors for persistence. Different models, different purposes.
4. **Model loaded as Resource.** Same pattern as `SectionEmbedder` — load once at startup, share across all processing, clean up on shutdown.
5. **Batch processing.** DJL supports batch inference. The service batches texts up to `batchSize` for efficiency, especially when embedding many sections or findings at once.
6. **Runtime dimension validation.** The `expectedDimension` config ensures model output matches the database schema. A misconfiguration is caught immediately, not on a failed INSERT.

## Acceptance Criteria

| Criterion | Test |
|-----------|------|
| `SemanticEmbeddingService.resource` loads model at startup | Resource acquired → predictor available |
| `SemanticEmbeddingService.resource` releases model on shutdown | Resource released → no memory leak |
| `embedBatch` produces vectors of `expectedDimension` | Config `expectedDimension = 768` → each vector has 768 elements |
| `embedBatch` raises on dimension mismatch | Model produces 384 but `expectedDimension = 768` → error raised |
| `embedBatch` handles empty input | Empty list → empty list returned |
| `embedBatch` chunks large inputs by `batchSize` | 100 texts, `batchSize = 32` → 4 DJL calls |
| `embedBatch` wraps DJL in `Async[F].blocking` | Verify blocking execution context |
| `embedBatch` raises `EmbeddingGenerationFailed` on DJL error | DJL throws → `EmbeddingGenerationFailed` with batch size |
| `embedSingle` delegates to `embedBatch` | Single text → `embedBatch(List(text))` |
| `embedFindings` extracts texts from `ConceptGroupPassOutput` findings | 3 outputs with 2 findings each → 6 texts extracted |
| `embedFindings` returns finding IDs paired with embeddings | 6 findings → 6 `(UUID, Array[Float])` tuples |
| Same text produces same embedding (deterministic) | Two calls with identical text → identical vectors |
| `modelPath` override loads from local path | `modelPath = Some("/tmp/model")` → loads from that path |
| Service shared by both pipelines | Same class used in decomposition and analysis entry points |
| `EmbeddingConfig` loads from PureConfig | Valid config → all fields including `expectedDimension` populated |
| `EmbeddingGenerationFailed` in shared package | Error type importable from `repcheck.analysis.embedding` |
