# Area 10.1 — LLM Provider Adapter

> Project: `llm-adapter` | Package: `repcheck.llm.adapter.core`, `repcheck.llm.adapter.providers`

## Purpose

Vendor-neutral LLM abstraction with provider-specific client implementations. Each provider (Claude, Ollama) has its own client that handles SDK/HTTP specifics, but all implement the common `LlmProvider[F]` trait. All calls return structured typed output — no raw text API.

---

## Classes

### `LlmProvider[F[_]]` — Core Trait (10.1)
**Package:** `repcheck.llm.adapter.core`

The vendor-neutral interface for LLM interactions. All pipeline code depends on this trait, never on a concrete provider.

```scala
trait LlmProvider[F[_]] {
  def completeStructured[A: Decoder](request: LlmRequest): F[A]
}
```

**Method: `completeStructured[A: Decoder](request: LlmRequest): F[A]`**

The single method for all LLM calls. Returns typed output, never raw text.

1. Accept `LlmRequest` with system prompt, user prompt, model tier, and optional JSON schema
2. Delegate to the concrete provider's transport (SDK or HTTP)
3. Receive raw text response as `LlmResponse` (internal)
4. Delegate to `StructuredOutputEnforcer` for JSON parsing into `A`
5. Return `F[A]` — typed, decoded output
6. On transport failure → raise provider-specific error (after retry at the 10.2 layer)
7. On parse failure → `StructuredOutputEnforcer` handles re-prompting and retries

> **Retry nesting order:** The `LlmRetryWrapper` (10.2) wraps the entire `completeStructured` call from the outside. Inside, the `StructuredOutputEnforcer` has its own re-prompt loop for parse failures. A transport failure during a re-prompt attempt will propagate up to the retry wrapper. The composition is: `retryWrapper(provider.completeStructured(request))` where `completeStructured` internally calls `enforcer.enforce(...)` which may re-call the transport.

> **Circe Decoder requirement:** All types used as `A` must have a Circe `Decoder[A]` instance. Per project conventions, use semi-auto derivation (`deriveDecoder`) in companion objects. This applies to: `SectionParseResult` (10.3), `SimplifiedConceptOutput` (10.4), `ConceptGroupPassOutput` (10.6), `Pass1RoutingScores`, `Pass2RoutingScores` (10.6), and any future structured output types.

### `LlmRequest` — Request Model (10.1)
**Package:** `repcheck.llm.adapter.core`

Immutable request object encapsulating all parameters for an LLM call. Used for both cloud API calls (Claude) and local sidecar calls (Ollama).

```scala
case class LlmRequest(
  systemPrompt: String,
  userPrompt: String,
  modelTier: ModelTier,
  maxTokens: Int,
  temperature: Double = 0.0,
  jsonSchema: Option[String] = None
)
```

### `LlmResponse` — Response Model (10.1)
**Package:** `repcheck.llm.adapter.core`

Immutable response object wrapping the LLM's output. Internal to providers — callers receive `F[A]`, not `F[LlmResponse]`.

```scala
case class LlmResponse(
  content: String,
  model: String,
  inputTokens: Int,
  outputTokens: Int,
  stopReason: StopReason
)
```

### `ModelTier` — Enum (10.1)
**Package:** `repcheck.llm.adapter.core`

Maps abstract tiers to concrete model identifiers per provider.

```scala
enum ModelTier {
  case Haiku, Sonnet, Opus, Local
}
```

- `Haiku`, `Sonnet`, `Opus` → Claude model identifiers (via `ClaudeProviderConfig.modelMappings`)
- `Local` → Ollama sidecar model (e.g., `llama3.2:1b`)

### `StopReason` — Enum (10.1)
**Package:** `repcheck.llm.adapter.core`

```scala
enum StopReason {
  case EndTurn, MaxTokens, StopSequence
}
```

### `StructuredOutputEnforcer[F[_]]` — Structured Output Parsing (10.1)
**Package:** `repcheck.llm.adapter.core`

Parses raw LLM text responses into typed Scala objects using Circe. Internal to the provider — callers never see raw text. The `maxRetries` value comes from `LlmAdapterConfig.structuredOutputMaxRetries` (10.2) and is passed at construction time by the entry point wiring.

```scala
class StructuredOutputEnforcer[F[_]: Async](
  maxRetries: Int
)
```

#### `enforce[A: Decoder](response: LlmResponse, retryFn: LlmRequest => F[LlmResponse], originalRequest: LlmRequest): F[A]`

1. Extract `content` from `LlmResponse`
2. Attempt `io.circe.parser.decode[A](content)`
3. If decode succeeds → return `A`
4. If decode fails and retries remaining > 0:
   - Construct corrective prompt: include original request + raw response + parse error message
   - Call `retryFn(corrective)` to re-prompt the LLM
   - Recursively attempt decode on new response (decrement retry counter)
5. If decode fails and retries exhausted → raise `StructuredOutputParseFailed(content, error.getMessage, maxRetries)`

> **Note:** The `retryFn` calls the provider's transport directly (SDK or HTTP) — these inner calls are NOT wrapped by `LlmRetryWrapper`. If a re-prompt transport call fails (e.g., 429), the error propagates up through the enforcer to the outer retry wrapper.

### `ClaudeProvider[F[_]]` — Claude Client Implementation (10.1)
**Package:** `repcheck.llm.adapter.providers`

Concrete `LlmProvider` implementation using the Anthropic Java SDK. Handles all Claude-specific SDK translation.

```scala
class ClaudeProvider[F[_]: Async](
  client: AnthropicClient,
  config: ClaudeProviderConfig,
  enforcer: StructuredOutputEnforcer[F]
) extends LlmProvider[F]
```

#### `completeStructured[A: Decoder](request: LlmRequest): F[A]`

1. Validate `request.modelTier` is not `Local` — raise `ProviderUnavailable("claude", "Local tier not supported")` if so
2. Resolve model ID: `config.modelMappings.get(request.modelTier)` → e.g., `"claude-3-haiku-20240307"`. Raise `ProviderUnavailable` if tier not in mappings.
3. Build `MessageCreateParams`:
   - `model` ← resolved model ID
   - `system` ← `request.systemPrompt`
   - `messages` ← single user message with `request.userPrompt`
   - `maxTokens` ← `request.maxTokens`
   - `temperature` ← `request.temperature`
4. Execute SDK call: `Async[F].blocking(client.messages.create(params))`
5. Translate SDK response to `LlmResponse`:
   - `content` ← first text block from response content
   - `model` ← response model identifier
   - `inputTokens` ← `response.usage.inputTokens`
   - `outputTokens` ← `response.usage.outputTokens`
   - `stopReason` ← map SDK stop reason to `StopReason` enum
6. Delegate to `enforcer.enforce[A](response, retryFn, request)` where `retryFn` re-executes steps 3–5 with corrective prompt
7. Return decoded `A`

### `OllamaProvider[F[_]]` — Ollama Client Implementation (10.1)
**Package:** `repcheck.llm.adapter.providers`

Concrete `LlmProvider` implementation for the Ollama sidecar. Handles HTTP communication with the local Ollama instance.

```scala
class OllamaProvider[F[_]: Async](
  httpClient: Client[F],
  config: OllamaProviderConfig,
  enforcer: StructuredOutputEnforcer[F]
) extends LlmProvider[F]
```

#### `completeStructured[A: Decoder](request: LlmRequest): F[A]`

1. Validate `request.modelTier` is `Local` — raise `ProviderUnavailable("ollama", "Only Local tier supported")` if not
2. Build Ollama `/api/generate` request JSON:
   - `model` ← `config.model` (e.g., `"llama3.2:1b"`)
   - `system` ← `request.systemPrompt`
   - `prompt` ← `request.userPrompt`
   - `stream` ← `false` (non-streaming)
   - `format` ← `"json"` (Ollama JSON mode)
3. Determine timeout: use `config.firstCallTimeoutSeconds` for the first call (model may be loading), `config.timeoutSeconds` for subsequent calls
4. Execute HTTP POST to `${config.baseUrl}/api/generate` via http4s client
5. Parse Ollama HTTP response body (JSON with fields `response`, `model`, `prompt_eval_count`, `eval_count`). Translate to `LlmResponse`:
   - `content` ← `response` field
   - `model` ← `config.model`
   - `inputTokens` ← `prompt_eval_count`
   - `outputTokens` ← `eval_count`
   - `stopReason` ← `StopReason.EndTurn` (Ollama doesn't distinguish stop reasons)
6. Delegate to `enforcer.enforce[A](response, retryFn, request)` where `retryFn` re-executes steps 2–5
7. Return decoded `A`

> **Ollama response DTO:** The raw Ollama HTTP response is parsed internally using a private `OllamaApiResponse` case class with Circe `Decoder`. This DTO is internal to `OllamaProvider` and not exposed.

## Key Design Decisions

1. **Structured output only.** No raw `complete()` method. Every LLM call in our pipeline expects typed output. The `StructuredOutputEnforcer` is internal to each provider — callers get `F[A]`, never `F[String]`.
2. **One trait, multiple clients.** `ClaudeProvider` wraps the Anthropic Java SDK. `OllamaProvider` wraps HTTP calls to localhost:11434. Each handles its own SDK/protocol specifics. Pipeline code depends only on `LlmProvider[F]`.
3. **ModelTier routes to provider.** `Local` → Ollama, `Haiku`/`Sonnet`/`Opus` → Claude. The pipeline entry point wires the correct provider for each use case.
4. **Anthropic Java SDK is synchronous.** All SDK calls wrap in `Async[F].blocking`. Ollama uses http4s which is already non-blocking.
5. **Re-prompting for parse failures.** Rather than immediately failing on malformed JSON, the enforcer gives the LLM one more chance with the error message. This significantly improves structured output reliability.
6. **Two-layer retry.** Transport retries (10.2 `LlmRetryWrapper`) wrap the entire call. Parse retries (`StructuredOutputEnforcer`) operate inside the provider. Inner re-prompt transport failures propagate to the outer retry layer.

## Acceptance Criteria

| Criterion | Test |
|-----------|------|
| `ClaudeProvider` translates `LlmRequest` to Anthropic SDK params | Mock SDK → verify `MessageCreateParams` fields match request |
| `ClaudeProvider` maps Haiku/Sonnet/Opus to correct model IDs | Request with `ModelTier.Haiku` → verify model ID from config mappings |
| `ClaudeProvider` rejects `Local` tier with `ProviderUnavailable` | Request with `ModelTier.Local` → `ProviderUnavailable` raised |
| `ClaudeProvider` wraps SDK call in `Async[F].blocking` | Verify blocking execution context used |
| `ClaudeProvider` translates SDK response to `LlmResponse` | Mock SDK returns response → verify `LlmResponse` fields |
| `OllamaProvider` builds correct `/api/generate` request | WireMock → verify POST body has `model`, `system`, `prompt`, `format: json` |
| `OllamaProvider` rejects cloud tiers (Haiku, Sonnet, Opus) | Request with `ModelTier.Haiku` → `ProviderUnavailable` raised |
| `OllamaProvider` uses generous timeout for first call | First call → verify `firstCallTimeoutSeconds` timeout applied |
| `OllamaProvider` uses standard timeout for subsequent calls | Second call → verify `timeoutSeconds` timeout applied |
| `OllamaProvider` parses Ollama response JSON correctly | WireMock returns Ollama JSON with `response`, `eval_count` → verify `LlmResponse` fields |
| `StructuredOutputEnforcer` decodes valid JSON | Valid JSON matching `Decoder[A]` → `A` returned |
| `StructuredOutputEnforcer` re-prompts on invalid JSON | Invalid JSON → corrective prompt sent → valid response decoded |
| `StructuredOutputEnforcer` raises after max retries | Invalid JSON after N retries → `StructuredOutputParseFailed` raised |
| Corrective prompt includes original request and parse error | Verify re-prompt contains raw response + error message |
| Inner re-prompt transport failure propagates to outer retry | Enforcer re-prompt hits 429 → error reaches `LlmRetryWrapper` |
| `LlmRequest` defaults temperature to 0.0 | No temperature specified → verify 0.0 sent to provider |
| `StopReason` correctly mapped from Anthropic SDK | SDK `end_turn` → `EndTurn`, SDK `max_tokens` → `MaxTokens` |
| Token counts populated from provider response | Claude: `usage.inputTokens`/`outputTokens`. Ollama: `prompt_eval_count`/`eval_count` |
