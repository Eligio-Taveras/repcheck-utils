# Area 10.3 — Ollama Provider Configuration

> Project: `llm-adapter` | Package: `repcheck.llm.adapter.providers`

## Purpose

Configuration and sidecar-specific behavior for the `OllamaProvider` implementation. The provider itself is defined in 10.1; this area covers the Ollama-specific configuration and section parse result types.

---

## Classes

### `OllamaProviderConfig` — Sidecar Config (10.3)
**Package:** `repcheck.llm.adapter.providers`

PureConfig auto-derived configuration for the Ollama sidecar.

```scala
case class OllamaProviderConfig(
  baseUrl: String = "http://localhost:11434",
  model: String = "llama3.2:1b",
  timeoutSeconds: Int,
  firstCallTimeoutSeconds: Int
)
```

- `baseUrl` — always localhost; Ollama runs as a sidecar in the same Cloud Run Job
- `firstCallTimeoutSeconds` — generous timeout (10-30s) for initial request when model is loading into memory
- `timeoutSeconds` — standard per-request timeout for subsequent calls (model already loaded)

> **No `maxRetries` field.** Transport retries for Ollama use the shared `LlmRetryWrapper` (10.2) with `LlmRetryConfig`, same as Claude. There is no Ollama-specific retry count.

### `SectionParseResult` — Section Parsing Output (10.3)
**Package:** `repcheck.llm.adapter.providers`

The structured output type that `OllamaProvider.completeStructured[List[SectionParseResult]]` returns when parsing bill text into sections. Requires Circe semi-auto `Decoder` in companion object.

```scala
case class SectionParseResult(
  sectionIndex: Int,
  sectionIdentifier: Option[String],
  heading: Option[String],
  content: String
)

object SectionParseResult {
  implicit val decoder: Decoder[SectionParseResult] = deriveDecoder
}
```

- `sectionIndex` — ordinal position (0-based) identifying sequence within the bill
- `sectionIdentifier` — e.g., "Sec. 101", "Title I" — present in structured bills, absent in plain text
- `heading` — section heading if present in the source text
- `content` — the full section text content

### Error Types (10.3)

> **`OllamaCallFailed` removed from `llm-adapter`.** The `llm-adapter` library is domain-agnostic and does not know about `billId`. Transport failures from `OllamaProvider` produce `LlmCallFailed` (10.2) like any other provider. If the decomposition pipeline needs domain-specific error wrapping, it should catch `LlmCallFailed` and wrap it in `DecompositionFailed` (10.4).

---

## Key Design Decisions

1. **Ollama is a LlmProvider implementation.** Not a separate client. It shares `LlmRequest`/`LlmResponse`, `StructuredOutputEnforcer`, and retry logic with `ClaudeProvider`. The only difference is HTTP transport instead of SDK.
2. **Localhost communication only.** Ollama runs as a sidecar in the same Cloud Run Job — no network hops, no authentication needed.
3. **Format-agnostic parsing.** The LLM handles any input format. No separate XML parser, text parser, or PDF parser needed. The system prompt instructs the model to identify section boundaries regardless of format.
4. **Generous first-call timeout.** Model loading on the first request can take 10-30 seconds. Separate config for first call vs. subsequent calls.
5. **Graceful degradation.** If section parsing fails entirely, the decomposition orchestrator treats the whole bill as one section. Analysis still proceeds.
6. **Domain-agnostic errors.** `llm-adapter` raises generic `LlmCallFailed`/`ProviderUnavailable`. Domain-specific wrapping (adding `billId`) happens in the consuming pipeline.

## Acceptance Criteria

| Criterion | Test |
|-----------|------|
| `OllamaProviderConfig` loads from config with defaults | Default config → `baseUrl` is `http://localhost:11434`, `model` is `llama3.2:1b` |
| `OllamaProviderConfig` allows override of all fields | Custom config → verify all fields overridden |
| `OllamaProviderConfig` has no `maxRetries` field | Transport retries via `LlmRetryWrapper` only |
| `SectionParseResult` decodes from valid JSON via Circe | `{"sectionIndex": 0, "heading": "Title I", "content": "..."}` → decoded |
| `SectionParseResult` handles absent optional fields | No `sectionIdentifier` or `heading` → `None` values |
| `SectionParseResult` preserves section ordering | List of 5 results → `sectionIndex` 0..4 in order |
| `SectionParseResult` Circe Decoder derived in companion | Companion object has `implicit val decoder: Decoder[SectionParseResult]` |
| First call timeout is longer than standard timeout | `firstCallTimeoutSeconds > timeoutSeconds` in default config |
| Section parsing handles XML-formatted bill text | XML bill text → sections identified by XML structure |
| Section parsing handles plain text bill | Plain text → sections identified by headings/numbering |
| Section parsing handles PDF-extracted text | Irregular whitespace/formatting → sections still identified |
| Transport failure produces `LlmCallFailed`, not domain error | Ollama 500 → `LlmCallFailed` raised (no `billId` in error) |
