# 4.2 Bill Repository & History

> Part of [Component 4 — Bills Pipeline Projects](../04-BILLS-PIPELINE.md) — Project A: `bill-metadata-pipeline`

## Overview

Doobie repositories for the `bills`, `bill_cosponsors`, and `bill_subjects` tables, plus archive-before-overwrite to their corresponding history tables. Uses `UpsertHelper` from `ingestion-common` (Component 3 §3.5) for SQL generation.

All table and column names come from `Tables.*` constants in `pipeline-models` (Component 2). All DOs and history DOs come from `shared-models` (Component 1).

## Classes

### `BillRepository[F[_]]` — Trait + Impl

```
trait BillRepository[F[_]] {
  def upsert(bill: BillDO): F[Unit]
  def findByBillId(billId: String): F[Option[BillDO]]
  def findByBillIds(billIds: List[String]): F[List[BillDO]]
  def findBillsNeedingTextCheck(): F[List[BillDO]]
  def updateTextFields(
    billId: String,
    textUrl: String,
    textFormat: String,
    textVersionType: String,
    textDate: Instant,
    latestTextVersionId: UUID
  ): F[Unit]
}
```

**`upsert`:**
- Uses `UpsertHelper.upsertFragment` with `Tables.Bills`
- Conflict column: `bill_id` (natural key)
- Updates all non-key, non-audit columns on conflict
- Sets `updated_at = NOW()` on conflict

**`findByBillId`:**
- Simple SELECT by `bill_id`
- Returns `Option[BillDO]` — `None` if not found
- Used by `BillMetadataProcessor` to fetch stored bill for `ChangeDetector` comparison

**`findBillsNeedingTextCheck`:**
- Returns bills where text needs checking per the text lifecycle:
  ```sql
  WHERE text_url IS NULL
     OR (text_version_type IS DISTINCT FROM 'ENR')
  ```
- Used by Project B (`bill-text-availability-checker`)

**`updateTextFields`:**
- Partial UPDATE: sets only `text_url`, `text_format`, `text_version_type`, `text_date`, `latest_text_version_id`, `updated_at`
- Does NOT overwrite other bill metadata columns
- Used by Project C (`bill-text-pipeline`) after downloading text

| Criterion | Test |
|-----------|------|
| `upsert` inserts new bill | Insert → `findByBillId` → returns bill with all fields |
| `upsert` updates existing bill on conflict | Insert → upsert with changed title → verify title updated |
| `upsert` does not create duplicate rows | Upsert same `billId` twice → exactly 1 row |
| `upsert` sets `updated_at` on conflict | Upsert existing → `updated_at` changed |
| `findByBillId` returns `None` for missing bill | Unknown ID → `None` |
| `findByBillId` returns stored bill with all fields | Insert → find → all fields match |
| `findByBillIds` returns matching bills | Insert 3 → find by 2 IDs → returns 2 |
| `findByBillIds` ignores missing IDs | Find by [existing, missing] → returns only existing |
| `findBillsNeedingTextCheck` returns bills without text | Bill with `textUrl = None` → included |
| `findBillsNeedingTextCheck` excludes enrolled bills | Bill with `textVersionType = "ENR"` → excluded |
| `findBillsNeedingTextCheck` includes non-final text bills | Bill with `textVersionType = "IH"` → included |
| `updateTextFields` sets text columns without touching metadata | Update text → verify title unchanged, text fields set |
| All SQL uses `Tables.*` constants | No hardcoded table/column name strings |

### `BillHistoryArchiver[F[_]]` — Trait + Impl

Archives the current state of a bill and its related rows before overwrite.

```
trait BillHistoryArchiver[F[_]] {
  def archiveBill(billId: String): F[UUID]
}
```

**`archiveBill`:**
1. Generates a `history_id` UUID (shared across all three history inserts for this archive event)
2. Copies current `bills` row to `bill_history` with `history_id` and `archived_at = NOW()`
3. Copies current `bill_cosponsors` rows to `bill_cosponsor_history` with same `history_id` and `archived_at`
4. Copies current `bill_subjects` rows to `bill_subject_history` with same `history_id` and `archived_at`
5. All three operations in a single transaction
6. Returns the `history_id`

The shared `history_id` links the bill, cosponsor, and subject snapshots for a single archive event, enabling reconstruction of the full bill state at any point in time.

Called by `BillMetadataProcessor` before upserting a changed bill. NOT called for new bills (no prior state to archive).

| Criterion | Test |
|-----------|------|
| `archiveBill` copies bill row to `bill_history` | Insert bill → archive → query `bill_history` → row exists with matching fields |
| `archiveBill` copies cosponsors to `bill_cosponsor_history` | Insert bill + 3 cosponsors → archive → 3 history rows |
| `archiveBill` copies subjects to `bill_subject_history` | Insert bill + 5 subjects → archive → 5 history rows |
| All three history inserts share the same `history_id` | Archive → all history rows have same `history_id` |
| `archived_at` is set to approximately NOW() | Archive → `archived_at` within 1 second of test time |
| Archive is transactional | Simulate failure mid-archive → no partial history rows |
| Bill with no cosponsors/subjects archives only the bill row | Bill with empty cosponsors/subjects → only `bill_history` row |
| Multiple archives produce distinct `history_id` values | Archive twice → two different `history_id` values |
| Returns the generated `history_id` | Verify return value matches the UUID in history rows |

### `BillCosponsorRepository[F[_]]` — Trait + Impl

```
trait BillCosponsorRepository[F[_]] {
  def replaceAll(billId: String, cosponsors: List[BillCosponsorDO]): F[Unit]
  def findByBillId(billId: String): F[List[BillCosponsorDO]]
}
```

**`replaceAll`:**
- Deletes existing cosponsors for the `bill_id`, then inserts the new set
- Wrapped in a single transaction
- Old rows are already archived by `BillHistoryArchiver` before this is called
- Caller is responsible for creating placeholder members for unknown cosponsor `member_id` values before calling this (FK constraint)

| Criterion | Test |
|-----------|------|
| `replaceAll` inserts cosponsors for new bill | Insert 3 cosponsors → `findByBillId` → returns 3 |
| `replaceAll` replaces cosponsors on re-ingest | Insert 3 → replaceAll with 2 different → returns 2 |
| `replaceAll` handles empty list (clears all) | replaceAll with empty → `findByBillId` → returns empty |
| Replace is transactional | Simulate failure mid-replace → old cosponsors still intact |
| FK constraint requires member to exist | Cosponsor with unknown `member_id` → FK violation error |

trait BillSubjectRepository[F[_]] {

Same replace pattern as cosponsors. Delete existing + insert new in one transaction.

| Criterion | Test |

| `replaceAll` replaces subjects on re-ingest | Insert 5 → replaceAll with 3 → returns 3 |

| Replace is transactional | Simulate failure mid-replace → old subjects still intact |
