# 4.4 Bill Text Availability API Client

> Part of [Component 4 — Bills Pipeline Projects](../04-BILLS-PIPELINE.md) — Project B: `bill-text-availability-checker`

## Overview

Fetches bill text version links from the Congress.gov text endpoint. This is a standalone client (not paginated) — the text endpoint returns all versions for a single bill in one response.

## Classes

### `BillTextApiClient[F[_]]`

```
class BillTextApiClient[F[_]: Async](
  config: CongressGovClientConfig,
  client: Client[F]
) {
  def fetchTextVersions(congress: Int, billType: String, number: String): F[BillTextLinksDTO]
}
```

**`fetchTextVersions` behavior:**
1. Builds URL: `{baseUrl}/bill/{congress}/{billType}/{number}/text?api_key={key}`
2. Makes HTTP GET request via the injected `Client[F]`
3. Decodes response using `BillTextLinksDTOGovSite` Circe decoder
4. Returns `BillTextLinksDTO` containing `List[TextVersion]`, each with `date: LocalDateTime` and `formats: List[Format]`
5. On HTTP 404: returns empty `BillTextLinksDTO` (bill exists but no text published yet — not an error)
6. On other HTTP errors: raises `BillTextCheckFailed`

**Retry and rate limiting:**
- Wraps calls in `RetryWrapper` with `CongressGovErrorClassifier` from `ingestion-common`
- HTTP 429/5xx → Transient (retry with backoff)
- HTTP 401/403 → Systemic (halt)
- HTTP 404 → not an error, returns empty result

**Migrates from:** `BillTextLinksApi` in `gov-apis` module. Key change: uses `ingestion-common` HTTP client and retry infrastructure.

### Error Types

```
final case class BillTextCheckFailed(
  billId: String,
  detail: String,
  cause: Option[Throwable] = None
) extends Exception(s"Failed to check text availability for bill $billId: $detail", cause.orNull)
```

### Acceptance Criteria

| Criterion | Test |
|-----------|------|
| Returns text versions with format URLs | WireMock: mock text endpoint → `BillTextLinksDTO` with versions |
| Handles 404 (no text available) gracefully | WireMock: 404 → empty `BillTextLinksDTO`, no exception |
| Returns multiple text versions when available | WireMock: response with 3 versions → all decoded |
| Preserves all format types (PDF, Formatted Text, XML) | Multiple formats in response → all decoded |
| `api_key` passed on every request | Inspect WireMock request → `api_key` present |
| Correct URL constructed from congress/billType/number | `(118, "hr", "1234")` → request to `/v3/bill/118/hr/1234/text` |
| HTTP 429 triggers retry | WireMock: first 429, second 200 → succeeds |
| HTTP 500 triggers retry | WireMock: first 500, second 200 → succeeds |
| HTTP 403 fails immediately | WireMock: 403 → `BillTextCheckFailed`, no retry |
| Network timeout raises `BillTextCheckFailed` | WireMock: delay exceeding timeout → error with detail |
| Malformed JSON raises descriptive error | WireMock: invalid JSON → error includes bill context |
