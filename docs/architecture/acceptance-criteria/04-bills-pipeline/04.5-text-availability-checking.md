# 4.5 Text Availability Checking Logic

> Part of [Component 4 — Bills Pipeline Projects](../04-BILLS-PIPELINE.md) — Project B: `bill-text-availability-checker`

## Overview

Scans tracked bills in AlloyDB for new or changed text versions, then emits `bill.text.available` events for each bill with new text. This is the bridge between metadata ingestion and text downloading.

## Classes

### `BillTextAvailabilityChecker[F[_]]`

```
class BillTextAvailabilityChecker[F[_]: Async](
  textApiClient: BillTextApiClient[F],
  billRepository: BillRepository[F],
  eventPublisher: IngestionEventPublisher[F],
  config: TextAvailabilityConfig,
  logger: PipelineLogger[F]
)
```

#### `checkAll(correlationId: UUID): Stream[F, ProcessingResult]`

Primary method. Returns a stream of per-bill results.

1. **Query candidates:** `billRepository.findBillsNeedingTextCheck()` → `List[BillDO]`
   - Includes all bills where `text_url IS NULL` or `text_version_type != 'ENR'`
   - No lookback window — scans all candidates. Heavy on cold start, light in steady state.
2. **Stream candidates** via `Stream.emits` then `parEvalMap(config.parallelism)`:
   - `checkBill(bill, correlationId)` per bill

#### `checkBill(bill: BillDO, correlationId: UUID): F[ProcessingResult]`

Per-bill text availability check.

1. **Fetch text versions:** `textApiClient.fetchTextVersions(bill.congress, bill.billType, bill.number)`
2. **Empty response** (no text available yet) → return `ProcessingResult.Skipped(billId, "no text available")`
3. **Select latest version:** Sort `TextVersion` list by `date` descending, take first
4. **Select best format** from the latest version, by priority:

   | Priority | Format | Rationale |
   |----------|--------|-----------|
   | 1 (best) | Formatted Text | Clean text, easiest for LLM consumption |
   | 2 | Formatted XML | Structured, can be parsed to text |
   | 3 | PDF | Requires extraction, may lose formatting |

   If no formats available in the version (edge case) → `ProcessingResult.Skipped`

5. **Compare against stored text:**
   - If `bill.textVersionType` is `None` → first text ever, this is new
   - If `bill.textVersionType` is `Some(code)` and differs from latest version's code → text updated
   - If same version code → `ProcessingResult.Skipped(billId, "same text version")`

6. **Emit event:**
   ```
   BillTextAvailableEvent(
     billId = bill.billId,
     congress = bill.congress,
     textUrl = selectedFormat.url,
     textFormat = selectedFormat.formatType.toString,
     versionCode = latestVersion.versionCode,
     previousVersionCode = bill.textVersionType
   )
   ```
   `eventPublisher.billTextAvailable(event)` → returns Pub/Sub message ID

7. Return `ProcessingResult.Succeeded(billId)`

#### Error Handling

- Per-bill failures are non-fatal. If text check fails for one bill (network error, parse error), the error is logged and `ProcessingResult.Failed` is returned. The stream continues. The failed bill will be retried on the next scheduled run.
- Event emission failure IS propagated — if we can't publish to Pub/Sub, the event is lost. This is handled by `RetryWrapper` around the publisher.

### Key Design Decisions

- **No text field updates here** — this checker only detects availability and emits events. Actual text field updates on the `bills` row happen in `bill-text-pipeline` (Project C) after text is downloaded.
- **Enrolled bills are terminal** — `textVersionType = "ENR"` means text is final. The bill is excluded from `findBillsNeedingTextCheck` so it's never checked again.
- **Format priority** — Formatted Text > XML > PDF. The downstream `BillTextDownloader` handles all three formats, but clean text is preferred for LLM consumption.
- **Full scan, no lookback** — queries all bills needing text, not just recently changed ones. On first deployment this may return thousands of bills. In steady state, only a handful of bills will have pending text.

### Acceptance Criteria

| Criterion | Test |
|-----------|------|
| Bill with no text → text found → event emitted | `textUrl = None`, API returns versions → event with `previousVersionCode = None` |
| Bill with no text → no text available → skipped | `textUrl = None`, API returns empty → `Skipped` |
| Bill with existing text → same version code → skipped | Stored `"IH"`, API returns `"IH"` → `Skipped` |
| Bill with existing text → new version code → event emitted | Stored `"IH"`, API returns `"EH"` → event with `previousVersionCode = Some("IH")` |
| Enrolled bill excluded from candidates | `textVersionType = "ENR"` → not in `findBillsNeedingTextCheck` results |
| Latest text version selected by date | Two versions: Jan 1 and Feb 1 → Feb 1 selected |
| Best format selected by priority | Version has PDF + Formatted Text → Formatted Text URL in event |
| Format with only PDF selected when no better option | Only PDF available → PDF URL in event |
| Version with no formats is skipped | Version with empty formats list → `Skipped` |
| Event payload includes correct `billId`, `congress`, `textUrl`, `textFormat` | Inspect emitted event fields |
| Event payload includes `versionCode` from text version | API returns version `"EH"` → event `versionCode = "EH"` |
| Per-bill failure does not halt stream | First bill's text check fails → second bill still checked |
| Failed bills produce `ProcessingResult.Failed` | Text check error → `Failed` with error detail |
| `parEvalMap` uses configured parallelism | Config parallelism = 4 → concurrent execution |
| All events use correlation ID from run context | Verify `correlationId` in published event envelope |
| Summary counts events emitted | 3 events published → `eventsEmitted = 3` |
