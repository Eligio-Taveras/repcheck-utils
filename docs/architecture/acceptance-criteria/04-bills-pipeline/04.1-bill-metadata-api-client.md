# 4.1 Bill Metadata API Client

> Part of [Component 4 — Bills Pipeline Projects](../04-BILLS-PIPELINE.md) — Project A: `bill-metadata-pipeline`

## Overview

Extends `CongressGovPaginatedClient[F, BillListItemDTO]` from `ingestion-common` (Component 3 §3.1) for bill-specific endpoints. Inherits pagination, rate limiting, and retry for free.

## Classes

### `BillsApiClient[F[_]]`

Congress.gov bill API client with two capabilities: paginated list and per-bill detail fetch.

```
class BillsApiClient[F[_]: Async](
  config: CongressGovClientConfig,
  client: Client[F]
) extends CongressGovPaginatedClient[F, BillListItemDTO] {

  override def fetchPage(params: FetchParams): F[PagedResponse[BillListItemDTO]]
  override def fetchDetail(detailUrl: String): F[BillDetailDTO]
}
```

**`fetchPage` behavior:**
1. Builds URL: `{baseUrl}/bill?api_key={key}&offset={offset}&limit={pageSize}&sort={sort}`
2. Appends `fromDateTime` and `toDateTime` if provided (ISO-8601 format: `yyyy-MM-dd'T'HH:mm:ss'Z'`)
3. Decodes response JSON using `BillListItemDTO` Circe decoder (via the `LegislativeBillDTOGovSite` external shape)
4. Extracts pagination metadata from response to build `PagedResponse`

**`fetchDetail` behavior:**
1. Calls the bill detail URL (from list item's `url` field, e.g., `/v3/bill/118/hr/1234`)
2. Appends `api_key` query parameter
3. Decodes full `BillDetailDTO` including sponsors, cosponsors, subjects, summaries, actions
4. Returns the detail DTO for `BillDetailDTO.toDO` conversion

**Inherited from `CongressGovPaginatedClient`:**
- `fetchAll(params)` — paginated stream via `tailRecM`, emitting `BillListItemDTO` items. Stops when `items.size < pageSize`.
- Rate limiting via `pageDelay` between page requests
- Retry via `RetryWrapper` with `CongressGovErrorClassifier`

**Migrates from:** `LegislativeBillsApi` in `gov-apis` module. Key change: extends `CongressGovPaginatedClient` instead of the old `PagingApiBase`, uses `FetchParams` / `PagedResponse` types.

### Error Types

```
final case class BillFetchFailed(
  endpoint: String,
  statusCode: Int,
  detail: String
) extends Exception(s"Failed to fetch bills from $endpoint (HTTP $statusCode): $detail")
```

Error classification delegated to `CongressGovErrorClassifier` from `ingestion-common`:
- HTTP 429/500/502/503 → `Transient` (retry)
- HTTP 401/403 → `Systemic` (halt)

### Acceptance Criteria

| Criterion | Test |
|-----------|------|
| `fetchPage` returns decoded bill list items | WireMock: mock `/v3/bill` → verify items decoded correctly |
| `fetchPage` passes `fromDateTime`/`toDateTime` as ISO query params | Inspect WireMock request → query contains `fromDateTime=2024-01-01T00:00:00Z` |
| `fetchPage` passes `api_key` on every request | Inspect WireMock request → `api_key` present |
| `fetchAll` paginates across multiple pages | WireMock: 3 pages (250, 250, 50) → 550 items collected |
| `fetchAll` stops when page has fewer items than `pageSize` | WireMock: page with 100 items, pageSize=250 → no further requests |
| `fetchDetail` decodes full bill detail with sponsors and subjects | WireMock: mock detail endpoint → `BillDetailDTO` with sponsors, cosponsors, subjects |
| `fetchDetail` appends `api_key` to detail URL | Inspect WireMock request → `api_key` present on detail call |
| HTTP 429 triggers retry | WireMock: first request 429, second 200 → succeeds |
| HTTP 500 triggers retry (classified Transient) | WireMock: first request 500, second 200 → succeeds |
| HTTP 403 fails immediately (classified Systemic) | WireMock: 403 → `BillFetchFailed` raised, no retry |
| Malformed JSON response raises descriptive error | WireMock: invalid JSON → error includes endpoint URL |
| Request timeout raises error | WireMock: delay exceeding timeout → timeout error |
