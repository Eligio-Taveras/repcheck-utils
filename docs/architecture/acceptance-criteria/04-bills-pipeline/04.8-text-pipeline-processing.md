# 4.8 Text Pipeline Processing

> Part of [Component 4 â€” Bills Pipeline Projects](../04-BILLS-PIPELINE.md) â€” Project C: `bill-text-pipeline`

## Overview

Event-driven pipeline that receives `bill.text.available` events, downloads bill text content, stores it in `bill_text_versions`, updates the `bills` row, and emits `bill.text.ingested` for downstream LLM analysis.

Unlike Projects A and B (which are scheduled), this pipeline is launched by the Launcher in response to Pub/Sub events.

## Classes

### `BillTextProcessor[F[_]]`

```
class BillTextProcessor[F[_]: Async](
  downloader: BillTextDownloader[F],
  textVersionRepository: BillTextVersionRepository[F],
  billRepository: BillRepository[F],
  eventPublisher: IngestionEventPublisher[F],
  config: BillTextPipelineConfig,
  logger: PipelineLogger[F]
)
```

#### `processEvent(event: BillTextAvailableEvent, correlationId: UUID): F[ProcessingResult]`

Processes a single `bill.text.available` event.

1. **Download text:**
   ```
   downloader.download(event.textUrl, event.textFormat) â†’ String
   ```
   On failure â†’ delegate to `PipelineFailureHandler` for retry/dead-letter

2. **Build text version DO:**
   ```
   BillTextVersionDO(
     versionId = UUID.randomUUID(),
     billId = event.billId,
     versionCode = TextVersionCode.fromString(event.versionCode),
     versionType = TextVersionCode.fullName(event.versionCode),
     versionDate = Instant.now,
     formatType = FormatType.fromString(event.textFormat),
     url = event.textUrl,
     content = Some(downloadedText),
     embedding = None,          // computed later by analysis pipeline
     fetchedAt = Some(Instant.now),
     createdAt = Instant.now
   )
   ```

3. **Store:**
   ```
   textVersionRepository.storeAndUpdateBill(versionDO) â†’ versionId
   ```
   Inserts text version row + updates `bills` row text fields in one transaction.

4. **Enrich with committee code (best effort):**
   ```
   billRepository.findByBillId(event.billId) â†’ Option[BillDO]
   ```
   Extract `committeeCode` if available. `None` if bill not found or no committee referral.

5. **Emit event:**
   ```
   BillTextIngestedEvent(
     billId = event.billId,
     versionId = versionDO.versionId,   // UUID PK of the just-created bill_text_versions row
     congress = event.congress,
     versionCode = event.versionCode,
     previousVersionCode = event.previousVersionCode,
     committeeCode = committeeCode     // Option[String], best-effort
   )
   ```
   `eventPublisher.billTextIngested(event)` â†’ returns Pub/Sub message ID

6. Return `ProcessingResult.Succeeded(event.billId)`

#### `processEvents(events: List[BillTextAvailableEvent], correlationId: UUID): Stream[F, ProcessingResult]`

Batch processing entry point. The Launcher may pass multiple events. Processes via `parEvalMap(config.parallelism)`.

```
Stream.emits(events).parEvalMap(config.parallelism)(event => processEvent(event, correlationId))
```

### Key Design Decisions

- **Event-driven** â€” only runs when there's work (a `bill.text.available` event). Launched by the Launcher via Cloud Run Jobs API.
- **Idempotent** â€” downloading and storing the same text version twice is safe. `bill_text_versions` gets a new row (different `version_id`), and `latest_text_version_id` on `bills` points to the most recent insert.
- **`committeeCode` is best-effort** â€” populated from the `bills` row if committee referral data exists. `None` if not available. Not a failure condition.
- **Download failure â†’ retry/dead-letter** â€” transient download failures are retried via `PipelineFailureHandler` (Component 3 Â§3.7). After max retries, the event goes to dead-letter for manual investigation.
- **Embedding deferred** â€” `BillTextVersionDO.embedding` is `None` at storage time. The Bill Analysis Pipeline computes embeddings after LLM analysis and back-fills them.

### Acceptance Criteria

| Criterion | Test |
|-----------|------|
| Downloads text from event's `textUrl` | Mock downloader â†’ verify `download` called with correct URL and format |
| Stores text version in `bill_text_versions` | Process event â†’ query `bill_text_versions` â†’ row exists with content |
| Updates `bills` row text fields | Process event â†’ query `bills` â†’ `text_url`, `text_format`, `latest_text_version_id` updated |
| Store and bill update are transactional | Simulate update failure â†’ no version row inserted |
| Emits `bill.text.ingested` event | Process event â†’ verify event published with correct payload |
| Event includes `versionCode` from incoming event | Incoming `versionCode = "EH"` â†’ outgoing event `versionCode = "EH"` |
| Event includes `previousVersionCode` from incoming event | Incoming `previousVersionCode = Some("IH")` â†’ propagated to outgoing |
| `committeeCode` populated when bill has committee data | Bill in DB with committee referral â†’ event `committeeCode = Some("HSAG")` |
| `committeeCode` is `None` when bill has no committee data | Bill with no committee â†’ event `committeeCode = None` |
| `committeeCode` is `None` when bill not found in DB | Unknown `billId` â†’ event `committeeCode = None`, no failure |
| Download failure triggers retry via `PipelineFailureHandler` | `TextDownloadFailed` â†’ `handleFailure` called |
| Permanent download failure â†’ `ProcessingResult.Failed` | Max retries exceeded â†’ `Failed` result |
| Batch processing uses `parEvalMap` | 3 events, parallelism = 2 â†’ concurrent processing |
| Single event failure does not halt batch | Event 1 fails, event 2 succeeds â†’ 1 Failed, 1 Succeeded |
| `versionId` returned from store matches `latest_text_version_id` on bill | Verify consistency after store |
| `embedding` is `None` in stored version | Query stored version â†’ `embedding IS NULL` |
| `fetchedAt` set to approximately now | Query stored version â†’ `fetchedAt` within 1 second of test time |
| Event includes `versionId` from stored version | `versionDO.versionId` â†’ outgoing event `versionId` matches |
| `versionId` in event is the UUID PK of `bill_text_versions` | Event versionId â†’ query `bill_text_versions` â†’ row exists |
