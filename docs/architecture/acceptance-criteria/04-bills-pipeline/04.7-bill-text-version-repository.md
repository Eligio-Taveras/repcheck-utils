# 4.7 Bill Text Version Repository

> Part of [Component 4 — Bills Pipeline Projects](../04-BILLS-PIPELINE.md) — Project C: `bill-text-pipeline`

## Overview

Doobie repository for the `bill_text_versions` table (defined in Component 1 §1.4) and partial updates to the `bills` row's text fields. Handles storing downloaded text content and keeping the `bills` row's latest-text reference in sync.

## Classes

### `BillTextVersionRepository[F[_]]` — Trait + Impl

```
trait BillTextVersionRepository[F[_]] {
  def insertVersion(version: BillTextVersionDO): F[UUID]
  def findByBillId(billId: String): F[List[BillTextVersionDO]]
  def findLatestByBillId(billId: String): F[Option[BillTextVersionDO]]
  def storeAndUpdateBill(version: BillTextVersionDO): F[UUID]
}
```

**`insertVersion`:**
- Inserts a new row into `bill_text_versions`
- Text versions are append-only (never updated or deleted)
- Each version gets a generated `version_id` UUID
- Returns the `version_id`

**`findByBillId`:**
- Returns all text versions for a bill, ordered by `version_date DESC` (latest first)
- Used for audit trail and diff analysis

**`findLatestByBillId`:**
- Returns the most recent text version for a bill
- Convenience: equivalent to `findByBillId(billId).headOption`

**`storeAndUpdateBill`:**
- The primary method used by `BillTextProcessor`. Performs two operations in a single transaction:
  1. `insertVersion(version)` → gets `versionId`
  2. `BillRepository.updateTextFields(billId, textUrl, textFormat, textVersionType, textDate, versionId)` → updates the denormalized text fields on the `bills` row to point to this new version
- Returns the `version_id`
- Transactional: if either operation fails, neither is committed

**`BillTextVersionDO` fields** (from Component 1 §1.4):
- `versionId: UUID` — PK, generated
- `billId: String` — FK to `bills.bill_id`
- `versionCode: TextVersionCode` — enum (IH, IS, RH, RS, EH, ES, ENR, etc.)
- `versionType: String` — full name (e.g., "Reported in House")
- `versionDate: Instant`
- `formatType: FormatType` — enum (formattedText, pdf, formattedXml)
- `url: String` — source URL
- `content: Option[String]` — full text content (populated by this pipeline)
- `embedding: Option[Array[Float]]` — vector embedding (populated later by analysis pipeline)
- `fetchedAt: Option[Instant]` — when content was downloaded
- `createdAt: Instant`

### Key Design Decisions

- **Append-only** — text versions are immutable once stored. If Congress.gov publishes a corrected version at the same legislative stage, it gets a new row with a later `version_date`.
- **Denormalized latest text on `bills`** — the `bills` row maintains a copy of the latest version's fields for fast queries. Updated every time a new version is stored via `storeAndUpdateBill`.
- **Embedding stored later** — `embedding` is `None` at insert time. The Bill Analysis Pipeline (downstream) computes and back-fills the embedding after LLM analysis.
- **Content nullable** — `content` is `Option[String]` because the schema supports versions where content hasn't been downloaded yet. In practice, this pipeline always populates it.

### Acceptance Criteria

| Criterion | Test |
|-----------|------|
| `insertVersion` creates row with generated UUID | Insert → query → row exists with non-null `version_id` |
| `insertVersion` stores all fields correctly | Insert with all fields populated → query → fields match |
| `insertVersion` preserves `content` text | Insert with 10KB text → query → content matches exactly |
| Append-only: multiple versions per bill allowed | Insert 3 versions for same bill → `findByBillId` returns 3 |
| `findByBillId` orders by `version_date` descending | 3 versions with different dates → latest first |
| `findByBillId` returns empty for unknown bill | Unknown `billId` → empty list |
| `findLatestByBillId` returns most recent version | 3 versions → returns the one with latest `version_date` |
| `findLatestByBillId` returns `None` for unknown bill | Unknown `billId` → `None` |
| `storeAndUpdateBill` inserts version AND updates bill text fields | Insert → check `bill_text_versions` row AND `bills.text_url` updated |
| `storeAndUpdateBill` is transactional | Simulate failure on bill update → version NOT inserted |
| `storeAndUpdateBill` sets `latest_text_version_id` on bill | After store → `bills.latest_text_version_id` = returned `version_id` |
| `embedding` is `None` at insert time | Insert without embedding → `embedding IS NULL` |
| All SQL uses `Tables.*` constants | No hardcoded table/column name strings |
