# 1.6 LLM Output Schemas

> Part of [Component 1 — repcheck-shared-models](../01-SHARED-MODELS.md)

### Classes (all new)

**Output schemas** (JSON structures returned by all LLM providers):

| Class | Fields |
|-------|--------|
| `BillSummaryOutput` | summary: String, readingLevel: String, keyPoints: List[String] |
| `TopicClassificationOutput` | topics: List[TopicScore] |
| `TopicScore` | topic: String, confidence: Double (0.0-1.0) |
| `StanceClassificationOutput` | stances: List[TopicStance] |
| `TopicStance` | topic: String, stance: StanceType, confidence: Double, reasoning: String |
| `PorkDetectionOutput` | findings: List[PorkFinding] |
| `PorkFinding` | porkType: PorkType, description: String, affectedSection: String, severity: ImpactSeverity |
| `ImpactAnalysisOutput` | impacts: List[ImpactItem] |
| `ImpactItem` | affectedGroup: String, impactType: String, description: String, severity: ImpactSeverity |
| `FiscalEstimateOutput` | estimatedCost: String, timeframe: String, assumptions: List[String], confidence: Double |
| `AlignmentScoreOutput` | topicScores: List[TopicAlignmentScore], overallScore: Double, highlights: List[AlignmentHighlight], reasoning: String |
| `TopicAlignmentScore` | topic: String, score: Double, explanation: String |
| `AlignmentHighlight` | billId: String, topic: String, stance: String, vote: String, alignment: Double |

**Supporting enums:**

| Enum | Values |
|------|--------|
| `StanceType` | Conservative, Progressive, Bipartisan, Neutral |
| `ImpactSeverity` | High, Medium, Low |
| `PorkType` | Earmark, Rider, UnrelatedProvision |

### Acceptance Criteria — 1.6 LLM Output Schemas

**Functional:**
1. Every output schema has Circe semi-auto derived `Decoder` and `Encoder`.
2. Every enum has a Circe codec that serializes to its lowercase string name and deserializes case-insensitively.
3. Every enum has a `fromString` parser returning `Either[String, EnumType]`.
4. `AlignmentScoreOutput` fields map to the columns in `score_history_highlights` (billId, topic, stance, vote, alignment).
5. `TopicScore.confidence`, `TopicStance.confidence`, and `TopicAlignmentScore.score` are all `Double` in 0.0-1.0 range.

**Error Handling:**
6. Decoders for output schemas produce clear errors when LLM returns malformed JSON (missing fields, wrong types).
7. Each unrecognized enum value returns a descriptive error including the unrecognized value and the set of valid values.

**Test Requirements:**
8. Round-trip codec test for every output schema with all fields populated.
9. Round-trip codec test for every enum value.
10. Decode-from-LLM test: sample JSON strings matching expected LLM output format, verify successful decode.
11. Malformed-LLM-output test: JSON missing required fields, verify descriptive `DecodingFailure`.
12. Case-insensitive enum test: "CONSERVATIVE", "conservative", "Conservative" all decode to `StanceType.Conservative`.
