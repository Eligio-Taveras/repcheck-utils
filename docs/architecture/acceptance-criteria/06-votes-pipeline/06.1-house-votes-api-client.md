# 6.1 House Votes API Client

> Part of [Component 6 — Votes Pipeline](../06-VOTES-PIPELINE.md)

## Overview

Extends `CongressGovPaginatedClient` for House vote endpoints. Fetches vote list items (paginated) and vote member positions (detail). The list endpoint provides metadata for change detection; the members endpoint provides per-member vote positions.

## Classes

### `HouseVotesApiClient[F[_]]`

```
class HouseVotesApiClient[F[_]: Async](
  config: CongressGovClientConfig,
  httpClient: Client[F],
  retryWrapper: RetryWrapper[F],
  logger: PipelineLogger[F]
) extends CongressGovPaginatedClient[F, VoteListItemDTO] {

  override def fetchPage(params: FetchParams): F[PagedResponse[VoteListItemDTO]]

  def fetchMembers(congress: Int, session: Int, voteNumber: Int): F[VoteMembersDTO]
}
```

**`fetchPage` behavior:**

1. Builds URL: `{baseUrl}/house-vote/{congress}/{session}?offset={offset}&limit={pageSize}&fromDateTime={fromDateTime}&toDateTime={toDateTime}&sort=updateDate+desc&format=json&api_key={apiKey}`
2. Decodes JSON response → `PagedResponse[VoteListItemDTO]`
3. Applies `config.pageDelay` between pages (handled by `fetchAll` in base trait)
4. On HTTP error: classifies via `CongressGovErrorClassifier`, retries transient errors via `retryWrapper`

**`fetchMembers` behavior:**

1. Builds URL: `{baseUrl}/house-vote/{congress}/{session}/{voteNumber}/members?format=json&api_key={apiKey}`
2. Decodes JSON response → `VoteMembersDTO` (includes `voteQuestion`, `results: List[VoteResultDTO]`)
3. This is the primary detail endpoint — it returns everything needed to construct `VoteDO` + `List[VotePositionDO]`

> **Why `fetchMembers` instead of `fetchDetail`?** The base trait's `fetchDetail(url: String): F[T]` returns the list item type `T` (`VoteListItemDTO`). But for votes, the detail we need is the members endpoint which returns `VoteMembersDTO` — a different type. So we define a separate method with the correct return type. The base trait's default `fetchDetail` is not overridden.

### Error Types

```
final case class HouseVoteFetchFailed(
  congress: Int,
  session: Int,
  voteNumber: Option[Int],
  detail: String,
  cause: Option[Throwable] = None
) extends Exception(
  s"Failed to fetch House vote (congress=$congress, session=$session${voteNumber.map(v => s", vote=$v").getOrElse("")}): $detail",
  cause.orNull
)
```

## Acceptance Criteria

| Criterion | Test |
|-----------|------|
| `fetchPage` returns paginated `VoteListItemDTO` list | Mock API returns page of votes → correctly decoded |
| `fetchAll` paginates until `items.size < pageSize` | Mock 3 pages (250, 250, 50) → all 550 items collected |
| `fetchMembers` returns `VoteMembersDTO` with positions | Mock members endpoint → DTO with results list decoded |
| `fetchMembers` includes `voteQuestion` field | Mock response with "On Passage" → field populated |
| API key included on every request | Inspect outgoing requests → `api_key` query param present |
| `fromDateTime` filter applied to list requests | Verify query string includes `fromDateTime` parameter |
| HTTP 429 classified as Transient and retried | Mock 429 → retried → succeeds on attempt 2 |
| HTTP 500/502/503 classified as Transient | Verify `CongressGovErrorClassifier.classify` returns Transient |
| HTTP 401/403 classified as Systemic (no retry) | Verify immediate failure |
| HTTP 404 for specific vote → `HouseVoteFetchFailed` | Mock 404 → error with congress/session/voteNumber context |
| Page delay respected between pages | 100ms delay, 3 pages → elapsed ≥ 200ms |
| Timeout applies per-request | Mock slow response → `TimeoutException` |
| `HouseVoteFetchFailed` includes congress, session, vote number context | Verify error message contains all identifiers |
