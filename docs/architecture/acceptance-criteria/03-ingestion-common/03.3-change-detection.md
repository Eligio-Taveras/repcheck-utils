# 3.3 Change Detection

> Part of [Component 3 — ingestion-common](../03-INGESTION-COMMON.md)

### New Classes

Generic change detection that works on any case class (via `Product`). Combines a fast `updateDate` pre-filter with full field-by-field diffing. The caller fetches the stored entity from its own repository and passes both versions in.

> **Standard strategy for all entity types:** This generic `ChangeDetector` is the standard change detection strategy for bills, members, amendments, committees, and any future entity types. Each pipeline calls `ChangeDetector.detect(incoming, stored, incomingUpdateDate, storedUpdateDate)` with its own entity DOs. The only documented exception is `VoteChangeDetector` (EXISTS query for position comparison), which lives in the vote-ingestion pipeline app because vote change detection requires a fundamentally different approach (comparing individual position records rather than a single entity's `updateDate`).

#### `ChangeDetector` — Object

```
object ChangeDetector {
  def detect[T <: Product](
    incoming: T,
    stored: Option[T],
    incomingUpdateDate: Instant,
    storedUpdateDate: Option[Instant]
  ): ChangeReport[T]
}
```

**Detection logic:**
1. If `stored` is `None` → `New(incoming)`
2. If `incomingUpdateDate < storedUpdateDate` → `Unchanged` + log warning (data regression, do not persist)
3. If `incomingUpdateDate == storedUpdateDate` → `Unchanged`
4. If `incomingUpdateDate > storedUpdateDate` → perform full field-by-field diff → `Updated(diffs)` or `Unchanged` (if dates differ but all fields identical)

**Field-by-field diffing** uses `Product` (case class built-in):
- `productElementNames` provides field names
- `productIterator` provides field values
- Recurses into nested case classes (also `Product`) and reports as `"sponsor.lastName"`
- For list fields containing items that implement `HasNaturalKey`, matches items by natural key and reports additions, removals, and per-item field diffs

#### `ChangeReport[T]` — Enum

```
enum ChangeReport[T] {
  case New(entity: T)
  case Updated(diffs: List[FieldDiff])
  case Unchanged
}
```

#### `FieldDiff`

```
case class FieldDiff(fieldName: String, oldValue: Any, newValue: Any)
```

For nested fields: `fieldName = "sponsor.lastName"`.
For list changes: `fieldName = "cosponsors[B000944].party"` (keyed by natural key), or `fieldName = "cosponsors[+B000944]"` (added), `fieldName = "cosponsors[-B000944]"` (removed).

#### `HasNaturalKey` — Trait

```
trait HasNaturalKey {
  def naturalKey: String
}
```

DOs that appear as items in list fields implement this trait so the differ can match items across old and new lists by key instead of by position.

| Acceptance Criterion | Test |
|---------------------|------|
| `stored = None` → `New(incoming)` | No prior record → `ChangeReport.New` |
| `incoming > stored` with field changes → `Updated(diffs)` | Change title → `Updated(List(FieldDiff("title", old, new)))` |
| `incoming > stored` with no field changes → `Unchanged` | Dates differ but fields identical → `Unchanged` |
| `incoming == stored` → `Unchanged` | Same timestamps → `Unchanged` |
| `incoming < stored` → `Unchanged` + warning log | Data regression → `Unchanged`, logs warning |
| Nested case class diffing | Change `sponsor.lastName` → diff reports `"sponsor.lastName"` |
| List addition detected by natural key | New cosponsor added → diff reports addition |
| List removal detected by natural key | Cosponsor removed → diff reports removal |
| List item modification detected | Cosponsor party changed → diff reports `"cosponsors[B000944].party"` |
| List ordering does not affect comparison | Same items in different order → no diff |
| Placeholder vs full entity produces diffs for all populated fields | Placeholder (mostly null) diffed against full entity → `Updated` with many diffs |
