# 3.4 Event Publishing

> Part of [Component 3 — ingestion-common](../03-INGESTION-COMMON.md)

### New Classes

Typed convenience layer over `PipelineEvent[T]` and `PubSubPublisher[F]` from `pipeline-models`. Provides one method per event type from BEHAVIORAL_SPECS §4 so pipeline apps emit events with correct types and fields.

#### `IngestionEventPublisher[F[_]]` — Trait

```
trait IngestionEventPublisher[F[_]] {
  def billTextAvailable(event: BillTextAvailableEvent): F[String]
  def billTextIngested(event: BillTextIngestedEvent): F[String]
  def voteRecorded(event: VoteRecordedEvent): F[String]
  def memberUpdated(event: MemberUpdatedEvent): F[String]
}
```

Each method:
1. Wraps the payload in `PipelineEvent.create(eventType, source, payload)`
2. Delegates to `PubSubPublisher[F].publish`
3. Returns the Pub/Sub message ID

Event types use the string constants: `"bill.text.available"`, `"bill.text.ingested"`, `"vote.recorded"`, `"member.updated"`.

The source string is injected via config (e.g., `"bill-ingestion"`, `"vote-ingestion"`, `"member-profile-pipeline"`, `"lis-mapping-refresher"`).

| Acceptance Criterion | Test |
|---------------------|------|
| `billTextAvailable` publishes with eventType `"bill.text.available"` | Capture published event → verify eventType |
| `billTextIngested` publishes with eventType `"bill.text.ingested"` | Capture published event → verify eventType |
| `voteRecorded` publishes with eventType `"vote.recorded"` | Capture published event → verify eventType |
| `memberUpdated` publishes with eventType `"member.updated"` | Capture published event → verify eventType |
| Source string comes from config, not hardcoded | Two instances with different source configs → verify different source in events |
| Event envelope includes generated UUID and timestamp | Published event → eventId is non-null UUID, timestamp ≈ now |
| Publisher failure propagates as `PublishFailed` | Mock publisher throws → `PublishFailed` raised |
| Retry wraps publish calls | Mock 1 failure then success → succeeds on retry |
