### New/Extended Classes

#### `DatabaseConfig`

```
final case class DatabaseConfig(
  host: String,
  port: Int = 5432,
  database: String,
  username: String,
  password: String,
  maxConnections: Int = 10
)
```

| Acceptance Criterion | Test |
|---------------------|------|
| PureConfig auto-derivation loads from HOCON | Parse sample HOCON → verify all fields |
| Defaults applied for port and maxConnections | Minimal HOCON → port=5432, maxConnections=10 |
| JDBC URL computed correctly | `host=db, port=5432, database=repcheck` → `jdbc:postgresql://db:5432/repcheck` |

#### `TransactorResource[F[_]]` — Object

Factory for creating a Doobie `HikariTransactor` from `DatabaseConfig`.

```
object TransactorResource {
  def make[F[_]: Async](config: DatabaseConfig): Resource[F, Transactor[F]]
}
```

| Acceptance Criterion | Test |
|---------------------|------|
| Transactor connects to PostgreSQL | `DockerPostgresSpec` → run simple query → succeeds |
| HikariCP pool size matches config | `maxConnections=5` → HikariCP maxPoolSize=5 |
| Resource release shuts down pool | After `use` block → pool is closed |

#### `UpsertHelper` — Object

Reusable SQL fragment builders for common upsert patterns.

```
object UpsertHelper {
  def upsertFragment(
    table: String,
    columns: List[String],
    conflictColumns: List[String],
    updateColumns: List[String]
  ): Fragment
}
```

This does NOT build full queries — it generates the `INSERT INTO ... ON CONFLICT ... DO UPDATE SET` SQL fragment. Each repository supplies its own values via Doobie parameter interpolation.

All table and column names are wrapped in **double-quoted identifiers** (e.g., `"bills"`, `"bill_id"`) to prevent SQL injection. Even though these values should only come from `Tables.*` constants and developer-defined lists, quoting ensures they are always treated as identifiers, never as SQL syntax.

| Acceptance Criterion | Test |
|---------------------|------|
| Generated SQL has correct INSERT structure | 3 columns → `INSERT INTO "table" ("a", "b", "c") VALUES (?, ?, ?)` |
| ON CONFLICT targets specified columns | conflictColumns = ["id"] → `ON CONFLICT ("id")` |
| DO UPDATE SET includes only update columns | updateColumns = ["name", "date"] → `DO UPDATE SET "name" = EXCLUDED."name", "date" = EXCLUDED."date"` |
| Table and column names are double-quoted identifiers | All identifiers wrapped in `"..."` to prevent SQL injection |
| Names containing special characters are safely quoted | Input `"bills; DROP TABLE users--"` → treated as identifier, not SQL |
| Table name uses constants (not hardcoded strings) | All callers use `Tables.*` constants |

*Note: `VoteHistoryArchiver` is vote-specific and lives in the vote-ingestion pipeline app, not in `ingestion-common`.*
