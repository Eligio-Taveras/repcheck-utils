# 3.6 Placeholder Entity Creation

> Part of [Component 3 — ingestion-common](../03-INGESTION-COMMON.md)

### New Classes

When a pipeline encounters a cross-entity reference to an entity not yet ingested (e.g., a bill references a sponsor whose `bioguide_id` is not in the `members` table), it creates a **placeholder row** with only the natural key populated. The owning pipeline fills in the full data later via normal upsert + `ChangeDetector` diff.

This replaces the `unresolved_references` table approach originally in Component 2 §2.9 (now removed).

**How it works:**

1. Bill ingestion encounters sponsor `B000944` → checks `members` table → not found
2. Creates placeholder: `MemberDO(bioguideId = "B000944")` with all other fields null/default
3. Bill's sponsor FK points to `B000944` immediately — no broken references
4. Member ingestion runs later → fetches full data for `B000944` → `ChangeDetector.detect(incoming, stored)` sees nearly every field changed → upserts full data

This pattern applies to **all entity types** — any pipeline that references another entity type creates a placeholder if the target doesn't exist.

#### `HasPlaceholder[T]` — Trait (lives in `repcheck-shared-models`)

> **Important**: This trait is defined in `repcheck-shared-models` (Component 1), package `repcheck.shared.models.placeholder`, NOT in `ingestion-common`. This avoids a circular dependency — shared-models has no repcheck dependencies, so the trait and its givens can coexist in the same library. `ingestion-common` imports it.

```
trait HasPlaceholder[T] {
  def placeholder(naturalKey: String): T
}
```

Each DO that can be referenced cross-pipeline implements this trait. The `placeholder` method returns a minimal valid instance with only the natural key populated.

```
// Given instances (in shared-models, on the DO companion objects):
given HasPlaceholder[MemberDO] with {
  def placeholder(naturalKey: String): MemberDO =
    MemberDO(bioguideId = naturalKey, /* all other fields null/default */)
}

given HasPlaceholder[BillDO] with {
  def placeholder(naturalKey: String): BillDO =
    BillDO(billId = naturalKey, /* all other fields null/default */)
}
```

#### `PlaceholderCreator[F[_]]` — Trait

Generic service that checks for existence and creates placeholders when needed.

```
trait PlaceholderCreator[F[_]] {
  def ensureExists[T <: Product: HasPlaceholder](
    naturalKey: String,
    repository: EntityRepository[F, T]
  ): F[Unit]
}
```

**`EntityRepository[F, T]`** — minimal trait that entity repositories implement:

```
trait EntityRepository[F[_], T] {
  def insertIfNotExists(entity: T): F[Unit]  // INSERT ... ON CONFLICT DO NOTHING
}
```

`ensureExists` behavior:
1. Attempts `INSERT ... ON CONFLICT DO NOTHING` with `HasPlaceholder[T].placeholder(naturalKey)`
2. If the row already exists (whether placeholder or full data), the insert is silently discarded
3. If the row does not exist, the placeholder is inserted

> **Why `ON CONFLICT DO NOTHING` instead of check-then-insert:** A naive `exists → insert` sequence has a race condition — if a concurrent pipeline upserts the real entity between the check and the insert, the placeholder would overwrite it with null fields. `ON CONFLICT DO NOTHING` is atomic: it either inserts the placeholder (if no row exists) or silently skips (if the real data already landed). This guarantees that a placeholder never overwrites richer data.

| Acceptance Criterion | Test |
|---------------------|------|
| `ensureExists` is a no-op when entity already exists | Member in DB → row unchanged (ON CONFLICT DO NOTHING) |
| `ensureExists` creates placeholder when entity missing | Member not in DB → placeholder inserted |
| `ensureExists` does not overwrite richer data on concurrent upsert | Real data inserted concurrently → placeholder silently discarded |
| Placeholder has only natural key populated | Created placeholder → all non-key fields are null/default |
| FK references to placeholder are valid | Bill with sponsor FK → placeholder member row → FK constraint satisfied |
| Owning pipeline upserts over placeholder with full data | Member ingestion runs → `ChangeDetector` sees diffs → full upsert |
| `ChangeDetector` correctly diffs placeholder vs full entity | Placeholder (mostly null) vs full DO → `Updated` with diffs for all populated fields |
| Idempotent: calling `ensureExists` twice is safe | Two calls for same key → only one row |
| Works for all entity types | Test with `MemberDO`, `BillDO`, `CommitteeDO` placeholders |
