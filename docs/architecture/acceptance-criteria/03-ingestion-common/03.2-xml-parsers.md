# 3.2 Senate/House XML Feed Parsers

> Part of [Component 3 — ingestion-common](../03-INGESTION-COMMON.md)

### New Classes

Congress.gov JSON API does not cover all data. Senate votes and member/committee data come from XML feeds hosted by senate.gov and clerk.house.gov. This area provides base XML parsing infrastructure — entity-specific mappings live in each pipeline.

#### `XmlFeedClient[F[_]]` — Trait

```
trait XmlFeedClient[F[_]] {
  def fetchXml(url: String): F[scala.xml.Elem]
  def fetchXmlStream(url: String): Stream[F, Byte]
}
```

Implementation wraps http4s client with `http4s-scala-xml` for XML parsing:
- No API key (public feeds)
- Retry via configurable `RetryConfig`
- Response validation (content-type check, non-empty body)
- UTF-8 decoding
- Uses `http4s-scala-xml` `EntityDecoder[F, scala.xml.Elem]` for type-safe XML response handling

| Acceptance Criterion | Test |
|---------------------|------|
| Fetches and parses valid XML | Mock XML response → returns `scala.xml.Elem` |
| Non-XML response raises `XmlParseFailed` | Mock HTML response → error |
| Empty response raises `XmlParseFailed` | Mock empty 200 → error |
| HTTP errors trigger retry | Mock 503 then 200 → succeeds on retry |
| Timeout applies per request | Mock slow response → `TimeoutException` |

#### `SenateXmlUrls` — Object

Constants for Senate XML feed URL patterns:
- `rollCallVote(congress, session, voteNumber)` → `https://www.senate.gov/legislative/LIS/roll_call_votes/vote{congress}{session}/vote_{congress}_{session}_{number}.xml`
- `rollCallIndex(congress, session)` → index URL for listing available votes
- `memberContactInfo` → `https://www.senate.gov/general/contact_information/senators_cfm.xml`
- `committeeMemberData` → `https://www.senate.gov/legislative/LIS_MEMBER/cvc_member_data.xml`
- `senatorLookup` → `https://www.senate.gov/about/senator-lookup.xml` (primary LIS-to-bioguide mapping source)

| Acceptance Criterion | Test |
|---------------------|------|
| URL patterns produce correct URLs | `rollCallVote(118, 1, 42)` → expected URL string |
| All URL functions return valid HTTPS URLs | Each function → starts with `https://` |

#### `XmlParsingHelpers` — Object

Utility functions for common XML extraction patterns:
- `textOpt(node: Node, tag: String): Option[String]` — extract optional text content
- `text(node: Node, tag: String): Either[XmlFieldMissing, String]` — extract required text
- `intOpt(node: Node, tag: String): Option[Int]` — extract optional int
- `int(node: Node, tag: String): Either[XmlFieldMissing, Int]` — extract required int
- `dateOpt(node: Node, tag: String, format: DateTimeFormatter): Option[LocalDate]` — extract optional date
- `children(node: Node, tag: String): Seq[Node]` — get child elements by tag name

| Acceptance Criterion | Test |
|---------------------|------|
| `text` returns value for present tag | `<root><name>John</name></root>` → `Right("John")` |
| `text` returns `Left(XmlFieldMissing)` for absent tag | `<root></root>` with tag "name" → `Left(...)` |
| `textOpt` returns `None` for absent tag | Same → `None` |
| `int` parses valid integers | `<root><count>42</count></root>` → `Right(42)` |
| `int` returns error for non-integer text | `<root><count>abc</count></root>` → `Left(...)` |
| `dateOpt` parses dates with given formatter | `<root><date>2024-01-15</date></root>` → `Some(LocalDate)` |
| `children` returns matching child nodes | XML with 3 `<member>` children → `Seq` of length 3 |

#### Error Types

```
final case class XmlParseFailed(url: String, cause: Throwable)
    extends Exception(s"Failed to parse XML from $url", cause)

final case class XmlFieldMissing(parentTag: String, fieldTag: String)
    extends Exception(s"Required field '$fieldTag' missing in '$parentTag'")
```
