## Overview

Doobie repositories for members, terms, and party history â€” with archive-before-overwrite for `members` and `member_terms`. `member_party_history` is append-only.

## Classes

### `MemberRepository[F[_]]`

```
class MemberRepository[F[_]: Sync](
  transactor: Transactor[F]
) {
  def upsert(member: MemberDO): F[Unit]
  def findByMemberId(memberId: String): F[Option[MemberDO]]
  def findByMemberIds(memberIds: List[String]): F[List[MemberDO]]
  def existsWithLisMapping(memberId: String): F[Boolean]
  def findPlaceholders(): F[List[MemberDO]]
}
```

- `upsert` â€” INSERT ON CONFLICT UPDATE using `memberId` as natural key
- `findByMemberId` â€” single member lookup
- `findByMemberIds` â€” batch lookup for event emission decisions
- `existsWithLisMapping` â€” joins `members` and `lis_member_mapping` to check if a senator has a mapping; used to decide whether to emit `member.updated` for Senate members
- `findPlaceholders` â€” returns members where `updateDate` is null (created by bills pipeline placeholder pattern)

### `MemberHistoryArchiver[F[_]]`

```
class MemberHistoryArchiver[F[_]: Sync](
  transactor: Transactor[F]
) {
  def archiveMember(memberId: String): F[Unit]
}
```

**`archiveMember` behavior:**

1. Generate a `history_id` UUID
2. Copy current `members` row to `member_history` with `history_id` and `archived_at`
3. Copy all `member_terms` rows for this member to `member_term_history` with same `history_id`
4. Both operations in a single transaction
5. If member does not exist (e.g., new member), no-op

> `member_party_history` is append-only and does **not** participate in the archive cycle.

### `MemberTermRepository[F[_]]`

```
class MemberTermRepository[F[_]: Sync](
  transactor: Transactor[F]
) {
  def replaceAll(memberId: String, terms: List[MemberTermDO]): F[Unit]
  def findByMemberId(memberId: String): F[List[MemberTermDO]]
}
```

- `replaceAll` â€” delete existing terms for member + insert new terms in a single transaction
- Called after `MemberHistoryArchiver.archiveMember` preserves the old terms in history

### `MemberPartyHistoryRepository[F[_]]`

```
class MemberPartyHistoryRepository[F[_]: Sync](
  transactor: Transactor[F]
) {
  def appendNew(memberId: String, entries: List[MemberPartyHistoryDO]): F[Unit]
  def findByMemberId(memberId: String): F[List[MemberPartyHistoryDO]]
}
```

- `appendNew` â€” inserts only entries that don't already exist (matching on `memberId` + `partyName` + `startYear`). Append-only; never deletes or updates existing rows.

### Acceptance Criteria

| Criterion | Test |
|-----------|------|
| Upsert inserts new member | Insert â†’ `findByMemberId` returns member |
| Upsert updates existing member | Insert, upsert with changed fields â†’ fields updated |
| Upsert uses `memberId` as natural key | Two upserts with same `memberId` â†’ one row |
| `existsWithLisMapping` returns true when mapping exists | Insert member + LIS mapping â†’ true |
| `existsWithLisMapping` returns false when no mapping | Insert member only â†’ false |
| `findPlaceholders` returns members with null updateDate | Insert placeholder + full member â†’ only placeholder returned |
| Archive copies member to `member_history` | Archive â†’ `member_history` has row with matching fields |
| Archive copies terms to `member_term_history` | Archive member with 3 terms â†’ 3 `member_term_history` rows |
| Archive shares `history_id` across tables | Archive â†’ same UUID in `member_history` and all `member_term_history` rows |
| Archive is transactional | Simulate failure mid-archive â†’ no partial writes |
| Archive no-ops for non-existent member | Archive unknown memberId â†’ no error, no rows |
| `replaceAll` deletes old terms and inserts new | Replace 2 terms with 3 â†’ only 3 in table |
| `replaceAll` is transactional | Simulate failure after delete â†’ old terms preserved |
| `appendNew` inserts only new party history entries | Append 3 entries, append 2 overlapping + 1 new â†’ 4 total |
| `appendNew` matches on memberId + partyName + startYear | Same triple â†’ no duplicate row |
| Doobie `Read`/`Write` round-trip for all DOs | Insert + read â†’ identical fields |
