# 5.6 LIS Mapping Processing

> Part of [Component 5 â€” Members Pipeline Projects](../05-MEMBERS-PIPELINE.md) â€” Project B: `lis-mapping-refresher`

## Overview

Refresh logic for the LIS-to-bioguide mapping table. Fetches senator-lookup.xml, upserts mappings, and emits `member.updated` events for senators who just became "complete" (already exist in `members` table and just gained a LIS mapping).

## Classes

### `LisMappingProcessor[F[_]]`

```
class LisMappingProcessor[F[_]: Async](
  xmlClient: SenatorLookupXmlClient[F],
  mappingRepo: LisMappingRepository[F],
  memberRepo: MemberRepository[F],
  eventPublisher: IngestionEventPublisher[F],
  logger: PipelineLogger[F]
) {
  def refreshAll(correlationId: UUID): F[LisMappingRefreshResult]
}
```

### `LisMappingRefreshResult`

```
final case class LisMappingRefreshResult(
  totalParsed: Int,
  inserted: Int,
  updated: Int,
  eventsEmitted: Int
)
```

**`refreshAll` behavior:**

1. Fetch mappings via `xmlClient.fetchMappings()` â†’ `List[SenatorLookupXmlDTO]`
2. Convert each to `LisMemberMappingDO` (set `lastVerified` to now)
3. Upsert batch via `mappingRepo.upsertBatch` â†’ `List[UpsertResult]`
4. For each `Inserted` result:
   a. Check if senator already exists in `members` table via `memberRepo.findByMemberId(mapping.memberId)`
   b. If exists: emit `MemberUpdatedEvent(memberId)` â€” this senator just became "complete"
   c. If not exists: no event â€” the member-profile-pipeline will handle this when it ingests the senator
5. Log summary: total parsed, inserted, updated, events emitted
6. Return `LisMappingRefreshResult`

**Why only `Inserted` triggers events:**

- `Updated` means the mapping already existed â€” the senator was already "complete." Refreshing `lastVerified` doesn't change anything downstream.
- `Inserted` means this is a new mapping. If the senator already has a profile in the `members` table (e.g., ingested by member-profile-pipeline but event was skipped because no LIS mapping existed), then *now* the senator is complete and downstream consumers should be notified.
