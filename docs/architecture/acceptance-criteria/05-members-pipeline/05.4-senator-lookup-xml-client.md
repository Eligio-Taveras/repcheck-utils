# 5.4 Senator Lookup XML Client

> Part of [Component 5 ‚Äî Members Pipeline Projects](../05-MEMBERS-PIPELINE.md) ‚Äî Project B: `lis-mapping-refresher`

## Overview

Fetches and parses `senator-lookup.xml` from senate.gov. Extracts LIS-to-bioguide mappings for senators within a configurable congress lookback window.

## Classes

### `SenatorLookupXmlClient[F[_]]`

```
class SenatorLookupXmlClient[F[_]: Async](
  xmlFeedClient: XmlFeedClient[F],
  config: LisMappingConfig,
  logger: PipelineLogger[F]
) {
  def fetchMappings(): F[List[SenatorLookupXmlDTO]]
}
```

> Uses `XmlFeedClient` from Component 3 ¬ß3.2 (not raw `Client[F]`) for XML fetch and parse infrastructure.

**`fetchMappings` behavior:**

1. HTTP GET `https://www.senate.gov/about/senator-lookup.xml`
2. Parse XML response into `List[SenatorLookupXmlDTO]`
3. Filter to senators who served in any congress within `config.congressLookbackWindow` (e.g., last 5 congresses from `config.currentCongress`)
4. Return filtered list

**Filtering logic:**

A senator is included if any of their `serviceDates` entries has a `congress` value >= `config.currentCongress - config.congressLookbackWindow + 1`. This ensures we have LIS mappings for all senators who may have votes in recent congresses.

### `LisMappingConfig`

```
final case class LisMappingConfig(
  currentCongress: Int = 118,
  congressLookbackWindow: Int = 5,
  requestTimeout: FiniteDuration = 30.seconds
)
```

### Error Types

```
final case class LisMappingFetchFailed(
  detail: String,
  cause: Option[Throwable] = None
) extends Exception(
  s"Failed to fetch senator-lookup.xml: $detail",
  cause.orNull
)
```

### Acceptance Criteria

| Criterion | Test |
|-----------|------|
| Fetches and parses senator-lookup.xml | WireMock: serve sample XML ‚Üí returns `List[SenatorLookupXmlDTO]` |
| Extracts `lisId` and `bioguideId` correctly | Sample senator ‚Üí correct fields |
| Extracts `isCurrent` flag | Senator with `current="yes"` ‚Üí `isCurrent = true` |
| Extracts service dates | Senator with 3 congress entries ‚Üí 3 `ServicePeriodDTO` |
| Filters by congress lookback window | Window = 5, current = 118 ‚Üí includes congress 114-118, excludes 113 |
| Senator spanning window boundary included | Service in congress 114 and 112 ‚Üí included (114 is in window) |
| Historical senator outside window excluded | Last served congress 110, window starts at 114 ‚Üí excluded |
| HTTP 404 raises `LisMappingFetchFailed` | WireMock: 404 ‚Üí error |
| HTTP 500 triggers retry | WireMock: first 500, second 200 ‚Üí succeeds |
| Malformed XML raises `LisMappingFetchFailed` | WireMock: invalid XML ‚Üí error with parse detail |
| Empty XML produces empty list | WireMock: valid XML with no senators ‚Üí empty list |
