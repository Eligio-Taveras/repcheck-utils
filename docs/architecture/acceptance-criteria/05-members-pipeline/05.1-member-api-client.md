# 5.1 Member API Client

> Part of [Component 5 — Members Pipeline Projects](../05-MEMBERS-PIPELINE.md) — Project A: `member-profile-pipeline`

## Overview

Extends `CongressGovPaginatedClient` for the Congress.gov member list and detail endpoints. Fetches members for a configurable congress number.

## Classes

### `MembersApiClient[F[_]]`

```
class MembersApiClient[F[_]: Async](
  client: Client[F],
  config: CongressGovClientConfig,
  logger: PipelineLogger[F]
) extends CongressGovPaginatedClient[F, MemberListItemDTO]
```

**Methods:**

- `fetchPage(params: FetchParams): F[PagedResponse[MemberListItemDTO]]` — single page of members. `FetchParams` includes the congress number, offset, and limit per `CongressGovPaginatedClient` base trait (Component 3 §3.1).
- `fetchDetail(detailUrl: String): F[MemberDetailDTO]` — full member profile by detail URL

**Behavior:**

1. `fetchPage` calls `/v3/member/congress/{params.congress}` with pagination parameters from `FetchParams`
2. `fetchDetail` calls the member detail URL (from `MemberListItemDTO.url`) to get full profile including terms, party history, depiction
3. Inherits pagination loop from `CongressGovPaginatedClient.fetchAll` — stack-safe via `tailRecM`
4. Rate limiting via `config.pageDelay`
5. Retry via `CongressGovErrorClassifier`

### Error Types

```
final case class MemberFetchFailed(
  memberId: Option[String],
  detail: String,
  cause: Option[Throwable] = None
) extends Exception(
  s"Failed to fetch member${memberId.fold("")(id => s" $id")}: $detail",
  cause.orNull
)
```

### Acceptance Criteria

| Criterion | Test |
|-----------|------|
| Fetches paginated member list for a congress | WireMock: `/v3/member/congress/118` → returns `MemberListItemDTO` list |
| Handles multi-page responses | WireMock: 3 pages → all members collected |
| Fetches member detail by URL | WireMock: detail URL → returns `MemberDetailDTO` with terms, party history |
| Rate limiting between pages | Two pages with `pageDelay = 100ms` → at least 100ms between requests |
| HTTP 404 on detail raises `MemberFetchFailed` | WireMock: 404 → error with memberId context |
| HTTP 500 triggers retry | WireMock: first 500, second 200 → succeeds |
| Network timeout raises error | WireMock: delay exceeding timeout → `MemberFetchFailed` |
| Empty page signals end of pagination | WireMock: empty items list → stream completes |
