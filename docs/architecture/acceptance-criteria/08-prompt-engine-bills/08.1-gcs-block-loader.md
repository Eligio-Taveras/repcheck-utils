# 8.1 GCS Block Loader

> Part of [Component 8 — Prompt Engine Bills](../08-PROMPT-ENGINE-BILLS.md)

## Overview

Loads instruction blocks and prompt profiles from GCS at runtime. Supports version filtering (semver in filenames), caching, and local file fallback for development. All GCS operations are wrapped in `Sync[F]` per project conventions.

## Classes

### `BlockLoader[F[_]]` — Trait + Impl

```
trait BlockLoader[F[_]] {
  def loadBlock(stage: PromptStage, blockName: String, version: String): F[InstructionBlock]
  def loadAllBlocks(stage: PromptStage, version: String): F[List[InstructionBlock]]
  def loadBlocksByNames(blockNames: List[String], version: String): F[Map[String, InstructionBlock]]
}
```

**`loadBlock`:**
1. Construct GCS path: `{config.basePath}/blocks/{stage.lowercase}/{blockName}-v{version}.yaml`
2. Read and deserialize via `GcsClient.readJson[InstructionBlock]`
3. Validate: `block.stage == stage` (file is in the correct stage directory)
4. Validate: `block.weight` is in range 0.0-1.0
5. On failure: raise `BlockLoadFailed` with GCS path and parse error

**`loadAllBlocks`:**
1. List all files under `{config.basePath}/blocks/{stage.lowercase}/` matching version via `GcsClient.listVersioned`
2. Load and deserialize each file
3. Return all blocks for that stage at that version

**`loadBlocksByNames`:**
1. For each block name, determine the stage from the block's metadata (or scan all stage directories)
2. Load each block by name and version
3. Return a map of `blockName → InstructionBlock`
4. If any block name is not found: raise `BlockLoadFailed` listing all missing names

> **Why load by name without stage?** `ChainAssembler.assemble` receives a `PromptProfile` whose `StageConfig` entries reference block names. The assembler needs all referenced blocks regardless of stage. `loadBlocksByNames` loads them all in one call, so the assembler doesn't need to know which stage directory each block lives in.

### `ProfileLoader[F[_]]` — Trait + Impl

```
trait ProfileLoader[F[_]] {
  def loadProfile(profileName: String, version: String): F[PromptProfile]
  def listProfiles(version: String): F[List[String]]
}
```

**`loadProfile`:**
1. Construct GCS path: `{config.basePath}/profiles/{profileName}-v{version}.yaml`
2. Read and deserialize via `GcsClient.readJson[PromptProfile]`
3. Validate: all `StageConfig.blockNames` are non-empty
4. Validate: all `StageConfig.weight` values are in 0.0-1.0
5. On failure: raise `ProfileLoadFailed` with GCS path and parse error

**`listProfiles`:**
1. List files under `{config.basePath}/profiles/` matching version
2. Extract profile names from filenames (strip `-v{version}.yaml` suffix)

### `BillPromptEngineConfig`

```
final case class BillPromptEngineConfig(
  gcsBucket: String = "repcheck-prompt-configs",
  basePath: String = "bills",
  defaultVersion: String,                    // set by CI, e.g., "1.0.0"
  localFallbackPath: Option[String] = None,  // e.g., "prompt-configs/bills" for dev
  cache: CacheConfig = CacheConfig()
)

final case class CacheConfig(
  enabled: Boolean = true,
  ttl: FiniteDuration = 5.minutes,           // blocks/profiles cached in memory
  maxEntries: Int = 200
)
```

> **`defaultVersion`** is set by CI on release (injected as a build property). Each deployment pins a specific prompt version. The version can be overridden per-request by the analysis pipeline if it needs to test a new prompt version.

### Local File Fallback

When `localFallbackPath` is configured (development mode), the loader reads from the local filesystem instead of GCS. The file layout mirrors the GCS layout:

```
prompt-configs/bills/
├── blocks/
│   ├── system/base-legislative-analyst-v1.0.0.yaml
│   ├── lens/fiscal-lens-v1.0.0.yaml
│   └── ...
└── profiles/
    ├── pass1-extraction-v1.0.0.yaml
    └── ...
```

The loader selects the data source based on config:
- `localFallbackPath = Some(path)` → read from filesystem via `Sync[F].blocking`
- `localFallbackPath = None` → read from GCS via `GcsClient`

### Caching

Blocks and profiles are cached in memory after first load. The cache is a simple TTL-based map (e.g., Caffeine or a Cats Effect `Ref`-based implementation):

- Cache key: `"{type}:{name}:{version}"` (e.g., `"block:fiscal-lens:1.0.0"`, `"profile:pass1-extraction:1.0.0"`)
- Cache miss → load from GCS/filesystem → cache
- TTL expiry → next request reloads from source
- Cache can be disabled via config for testing

### Error Types

```
final case class BlockLoadFailed(
  path: String,
  detail: String,
  cause: Option[Throwable] = None
) extends Exception(s"Failed to load instruction block from $path: $detail", cause.orNull)

final case class ProfileLoadFailed(
  path: String,
  detail: String,
  cause: Option[Throwable] = None
) extends Exception(s"Failed to load prompt profile from $path: $detail", cause.orNull)
```

## Acceptance Criteria

| Criterion | Test |
|-----------|------|
| `loadBlock` reads and deserializes a block from GCS | Mock GCS client → returns valid YAML → `InstructionBlock` with correct fields |
| `loadBlock` validates stage matches directory | Block in `lens/` dir with `stage: "system"` → `BlockLoadFailed` |
| `loadBlock` validates weight in 0.0-1.0 | Block with `weight: 1.5` → `BlockLoadFailed` |
| `loadBlock` raises `BlockLoadFailed` for missing file | GCS returns not found → `BlockLoadFailed` with path |
| `loadBlock` raises `BlockLoadFailed` for malformed YAML | GCS returns invalid YAML → `BlockLoadFailed` with parse error |
| `loadAllBlocks` returns all blocks for a stage and version | 3 blocks in `lens/` at v1.0.0 → returns 3 |
| `loadAllBlocks` returns empty list for empty stage | No blocks in `custom/` → empty list |
| `loadBlocksByNames` returns map of all requested blocks | Request 3 names → returns map with 3 entries |
| `loadBlocksByNames` raises error if any name missing | Request 3 names, 1 missing → `BlockLoadFailed` listing the missing name |
| `loadProfile` reads and deserializes a profile from GCS | Mock GCS → valid profile YAML → `PromptProfile` with correct chain |
| `loadProfile` validates all weights in range | Profile with `weight: -0.1` → `ProfileLoadFailed` |
| `loadProfile` validates non-empty block names | `StageConfig` with empty `blockNames` → `ProfileLoadFailed` |
| `listProfiles` returns available profile names | 3 profiles in GCS → returns 3 names |
| Local fallback reads from filesystem when configured | `localFallbackPath = Some("...")` → reads from local file, not GCS |
| Local fallback disabled when path is None | `localFallbackPath = None` → reads from GCS |
| Cache returns cached block on second load | Load block → load again within TTL → GCS not called second time |
| Cache expires after TTL | Load block → wait > TTL → load again → GCS called again |
| Cache disabled via config | `cache.enabled = false` → every load hits GCS |
| PureConfig loads `BillPromptEngineConfig` from HOCON | Parse sample HOCON → all fields populated |
| Defaults applied when optional keys absent | Minimal HOCON (defaultVersion only) → verify defaults |
